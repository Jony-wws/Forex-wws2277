# Telegram Progress — прогресс-бар для долгих скриптов

Модуль `scripts/telegram_progress.py` показывает в Telegram **одно** сообщение
с прогресс-баром, которое обновляется по мере выполнения цикла через
`editMessageText`. Это удобнее, чем сыпать в чат пачку отдельных
сообщений «10%», «20%», «30%»…

## Как выглядит сообщение

```
⏳ Цикл 5ч · 2026-05-16 18:30 UTC+5

████░░░░░░  40%
Анализ пар: 11/28 · EURJPY

Прошло: 1 мин 12 сек
Осталось: ориентировочно 1 мин 48 сек
```

После завершения сообщение заменяется на:

```
✅ Цикл 5ч · 2026-05-16 18:30 UTC+5 — завершено
██████████  100%
Время выполнения: 3 мин 4 сек
```

…а ниже отдельными сообщениями приходит полный отчёт цикла.

## Формат прогресс-бара

Шкала из 10 ячеек (можно изменить через `bar_width`):

| Процент | Бар         |
|---------|-------------|
| 0%      | `░░░░░░░░░░` |
| 25%     | `███░░░░░░░` |
| 50%     | `█████░░░░░` |
| 75%     | `████████░░` |
| 100%    | `██████████` |

Заполненные ячейки — `█` (U+2588), пустые — `░` (U+2591).

## Переменные окружения

| Переменная             | Назначение                                    |
|------------------------|-----------------------------------------------|
| `TELEGRAM_BOT_TOKEN`   | Токен Telegram-бота (от `@BotFather`)         |
| `TELEGRAM_CHAT_ID`     | ID чата / канала, куда писать прогресс        |

Если переменные не заданы, модуль работает в **no-op** режиме: печатает
прогресс в `stdout` и не падает. Это удобно для локальных запусков и
юнит-тестов.

## Быстрый старт

```python
from telegram_progress import TelegramProgress

progress = TelegramProgress(title="5-часовой цикл")
progress.start("Подготовка к запуску...")

# ... ваш долгий код ...
progress.update(10, "Данные загружены")
# ...
progress.update(50, "Половина пройдена")
# ...
progress.update(90, "Готовим отчёт...")

# Финальный шаг: editMessageText заменит прогресс-бар на "завершено",
# и отдельно прилетит full_report (если задан).
progress.complete(full_report="<b>Итоговый отчёт</b>\n...")
```

## API

### Класс `TelegramProgress`

| Метод                              | Что делает                                                 |
|------------------------------------|------------------------------------------------------------|
| `TelegramProgress(title=..., bar_width=10)` | Создаёт объект, читает env-переменные             |
| `start(status="Запуск...")`        | Отправляет первое сообщение, запускает таймер              |
| `update(percent, status="")`       | `editMessageText` с новым процентом и статусом             |
| `complete(full_report=None, summary=None)` | Финал: 100% + опциональный длинный отчёт           |

### Функции форматирования (можно использовать отдельно)

| Функция                              | Возвращает                                    |
|--------------------------------------|-----------------------------------------------|
| `format_progress_bar(percent, width=10)` | Строка вида `████░░░░░░`                  |
| `format_time_elapsed(seconds)`       | `"2 мин 15 сек"`                              |
| `format_time_remaining(seconds)`     | `"ориентировочно 3 мин"` или `…рассчитывается`|

### Низкоуровневые функции (для своих сценариев)

| Функция                                                  | Что делает                              |
|----------------------------------------------------------|-----------------------------------------|
| `send_progress_message(token, chat_id, text)`            | `sendMessage`, возвращает `message_id`  |
| `update_progress_message(token, chat_id, message_id, text)` | `editMessageText`                     |
| `complete_progress_message(token, chat_id, message_id, summary, full_report=None)` | финальный шаг |

## Интеграция в другие скрипты

1. **Добавьте импорт:**
   ```python
   import sys, os
   sys.path.insert(0, os.path.dirname(os.path.abspath(__file__)))
   from telegram_progress import TelegramProgress
   ```

2. **Создайте экземпляр в начале `main()`:**
   ```python
   progress = TelegramProgress(title="Мой скрипт")
   progress.start("Старт...")
   ```

3. **Расставьте `update()` в ключевых точках:**
   ```python
   progress.update(10, "Данные загружены")
   # ... работа ...
   progress.update(50, "Половина готова")
   # ... работа ...
   progress.update(90, "Сохраняем результат")
   ```

4. **Замените финальный `send_telegram(text)` на `progress.complete(full_report=text)`.**

## Пример: `scripts/cycle_5h.py`

Реальная интеграция в 5-часовом цикле — посмотрите в коде:

- `progress.start(...)` в начале `main()`.
- `progress.update(10, ...)` — данные подготовлены к загрузке.
- `progress.update(30, ...)` — sweep по всем 28 парам завершён.
- `progress.update(60, ...)` — бэктест на 5d/30d/365d готов.
- `progress.update(80, ...)` — топ-3 выбран.
- `progress.update(90, ...)` — отчёт сформирован, отправляем.
- `progress.complete(full_report=text)` — 100% + полный отчёт.

Между фиксированными чекпойнтами модуль ещё «играет» прогрессом
поштучно во время `sweep_all_pairs()` — через `progress_cb`, который
тикает 28 раз (по одной паре) и плавно увеличивает процент от 10 до 30.

## ETA — как считается

Линейная экстраполяция:

```
elapsed = now - start_ts
total_estimated = elapsed * 100 / percent
remaining = total_estimated - elapsed
```

На малых процентах (≤ 5%) оценка нестабильна — поэтому при `percent <= 0`
функция возвращает «рассчитывается».

## Ограничения Telegram

- `editMessageText` не имеет жёсткого rate-limit per-chat, но Telegram
  не любит, когда одно сообщение редактируется чаще, чем раз в секунду.
  В `cycle_5h.py` тиков мало (6 чекпойнтов + 28 тиков от sweep =
  ≤ 35 обновлений за ~5 минут), это далеко от лимита.
- Лимит длины сообщения — 4096 символов. Поэтому `complete()` отправляет
  `full_report` **новыми** сообщениями (chunked по 3900 символов), а
  само прогресс-сообщение остаётся компактным.
- Если Telegram возвращает `message is not modified` (текст не изменился) —
  это игнорируется, не считается ошибкой.

## No-op режим

Если хотя бы одна из переменных `TELEGRAM_BOT_TOKEN` / `TELEGRAM_CHAT_ID`
не задана, модуль печатает в `stdout`:

```
[telegram_progress] TELEGRAM_BOT_TOKEN / TELEGRAM_CHAT_ID не заданы — прогресс будет печататься локально
[telegram_progress] start: 0% — Подготовка к запуску...
[telegram_progress] 10% — Данные загружены
[telegram_progress] 30% — Анализ пар завершён
...
```

Это специально, чтобы локальный запуск `python scripts/cycle_5h.py` не
требовал бота и не падал.
