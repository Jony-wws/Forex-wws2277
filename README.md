## Главное правило

**Не работай как чат-бот. Работай как исследователь.**

Чат-бот получает вопрос и сразу даёт один ответ. Исследователь получает **цель**
и работает циклами, пока не достигнет её — или честно не докажет, что она недостижима.

Этот режим называется **глубокая автономная работа (deep work / autonomous iterative research)**.

## Универсальный промпт (для запуска режима в любом новом чате)

```
Работай в режиме глубокой работы (deep work):
1. Цель: [конкретная измеримая цель]
2. Пробуй разные методы, не останавливайся на первом
3. Присылай промежуточные отчёты с цифрами
4. Работай честно: без подгонки, без заглядывания в будущее
5. Продолжай, пока не достигнешь цели или не докажешь, что она недостижима
```

## 5 шагов — подробно

### Шаг 1. Цель с измеримым критерием
Прежде чем начать — сформулируй цель так, чтобы можно было ЧИСЛОМ проверить,
достигнута она или нет.
- Плохо: «улучши стратегию»
- Хорошо: «winrate ≥65% каждый месяц за 12 месяцев бэктеста, ровно 3 сделки в день, экспирация 3 часа»

Зафиксируй также **ограничения**, которые нельзя нарушать (например: сделки
обязательны каждый день, нельзя пропускать; вход только после новостей).

### Шаг 2. Цикл гипотеза → тест → отчёт → следующая гипотеза
Не останавливайся на первом методе. Каждая итерация:
1. **Гипотеза** — что может улучшить результат (новый фильтр, другие признаки, ансамбль моделей, другие временные слоты…)
2. **Тест** — напиши код, прогони полный бэктест, получи цифры
3. **Сравнение** — стало лучше или хуже по ГЛАВНОМУ критерию? Сохрани результат каждого варианта
4. **Вывод** — что оставить, что отбросить, какая следующая гипотеза

Тестируй варианты ПАРАЛЛЕЛЬНО где возможно. Веди таблицу всех вариантов с результатами.
Неудачные варианты — это тоже результат: запиши их, чтобы не повторять.

### Шаг 3. Промежуточные отчёты
Регулярно показывай заказчику текущие цифры, не дожидаясь конца работы:
- какой вариант сейчас лучший и его метрики (по месяцам, не только среднее)
- что тестируется прямо сейчас
- что планируется дальше

Если результаты разных вариантов отличаются — объясни ПОЧЕМУ (это разные
параллельные эксперименты, а не «потеря» предыдущего результата).

### Шаг 4. Честность — без обмана данных
Это самое важное. Результат, полученный нечестно, бесполезен и опасен:
- **Никакого заглядывания в будущее (look-ahead bias)**: решение в момент T использует только данные ДО момента T
- **Walk-forward**: модель обучается только на прошлых месяцах, тестируется на следующем
- **Никакой подгонки под тест**: нельзя подбирать параметры по тестовому периоду и выдавать это за прогноз
- **Реальные данные**, без симуляторов
- Помни про **множественное тестирование**: если перебрал 20 вариантов и выбрал лучший — его результат на новых данных скорее всего будет ниже. Сообщай об этом риске.

### Шаг 5. Продолжай — или докажи невозможность
Работа заканчивается только в двух случаях:
1. **Цель достигнута** — и проверена честно
2. **Доказано, что цель недостижима** — с математическим объяснением
   (например: при 60 сделках в месяц даже система с истинным winrate 65% из-за
   биномиальной дисперсии (σ≈6%) будет проваливать ~половину месяцев; чтобы КАЖДЫЙ
   месяц показывать ≥65%, нужен истинный winrate ~75%+)

В обоих случаях — финальный отчёт: лучший результат, все протестированные методы,
честные ограничения и риски.

### Лучшая найденная система (V6b)

**ML-ансамбль из 7 глубоких моделей градиентного бустинга + фильтр новостей.**

### ⚠️ Честный вывод: почему НЕ 13/13 месяцев

Цель «≥65% КАЖДЫЙ месяц при обязательных 3 сделках в день» **математически
недостижима честным путём** при истинном winrate ~65%:

- В месяце ~60–69 сделок. Биномиальная дисперсия: σ ≈ √(0.65·0.35/63) ≈ **6%**
- Даже система с ИСТИННЫМ winrate 65% покажет <65% примерно в **половине месяцев**
  чисто из-за случайности
- Чтобы каждый месяц наблюдать ≥65%, нужен истинный winrate **~75–78%** — на
  честных walk-forward бэктестах форекс H1 такого уровня не существует
  (это уровень, при котором за год можно удвоить депозит многократно)
- Запрет пропусков сделок убирает главный инструмент повышения месячного
  winrate — фильтрацию слабых сигналов

**Риск множественного тестирования**: перебрано 15+ вариантов, выбран лучший —
на новых (живых) данных ожидаемый результат ниже бэктеста: реалистично **~60–64%**
за год. Это выше безубытка (55.6%), но требует дисциплины и контроля рисков.

### Дополнение 3: система E «FLEX» — без расписания, 28 пар, кросс-проверка на 2 источниках (13.06.2026)

По запросу JONY: убрать любое жёсткое расписание (система сама решает КОГДА
торговать), ровно 3 сделки в сутки, экспирация 3ч, исключить CADCHF (брокер
JONY её не даёт). Построена система **E**: чемпион+МТФ (M15+H1+H4+D1),
сканирует 28 пар (минус CADCHF) каждый час, выбирает 3 самых уверенных входа
за сутки по причинному (causal) расширяющемуся квантильному порогу уверенности
(q=0.80), при недоборе — дозаполнение до ровно 3/день. Все фичи причинные,
walk-forward по месяцам, фильтр новостей.

**Главное — честная кросс-проверка на ДВУХ независимых источниках цен**
(как два разных брокера): один и тот же список сделок переоценён и на Yahoo, и
на независимом histdata-фиде. Результаты совпадают:

| Источник цен | Год | n | Мес ≥65% | Худший (полный) мес |
|---|---|---|---|---|
| **Yahoo** | **74.3%** | 786 | 11/13 | 61.7% (нояб; июнь-26 неполный 40%/30) |
| **Независимый (histdata)** | **74.0%** | 670 | 10/12 | 63.2% (нояб) |

Помесячно (независимый источник): 06 80.0 · 07 71.0 · 08 89.1 · 09 79.3 ·
10 78.0 · 11 63.2 · 12 67.8 · 01 67.8 · 02 73.2 · 03 64.9 · 04 79.3 · 05 78.2
(июнь-26 без данных). Старый champion на этом же независимом фиде ~62% — то
есть рост E настоящий (~+10%), а не артефакт одного источника.

**Важная техническая находка:** первая кросс-проверка дала расхождение
(75%↔65%). Причина — НЕ в стратегии, а в данных: histdata-таймстемпы это
**локальное время Нью-Йорка с переходом на летнее время (DST)**, а не
фиксированный EST. Старый сбор использовал фиксированный +5ч → летом сдвиг на
1 час, цены входа/выхода брались из соседней свечи. После пересборки с
`America/New_York` (DST-aware) корреляция часовых движений двух фидов стала
0.89 на лаге 0 (было разбито между лагом −1 и 0) — фиды выровнены, и оба дают
~74%.

**Честные оговорки (реальные деньги):**
- E — лучший из ~15 проверенных вариантов; вживую обычно немного хуже бэктеста.
  Реалистично закладывать ~65–72%, а не гарантированные 74%.
- Сила самой модели тонкая: на ВСЕХ сигналах подряд точность лишь ~53.5%.
  Высокий процент — только за счёт отбора 3 самых уверенных сигналов в сутки.
- Преимущество сконцентрировано в окне 17:00–22:00 UTC+5; движения там
  небольшие (4–6 пунктов). В ликвидные дневные часы (5–16 UTC) edge исчезает
  (~52%).
- «≥65% каждый месяц» при жёстких 3 сделках/день статистически негарантируемо
  (σ≈6%/мес). Слабые месяцы будут (нояб ~62–63%).

**Статус:** чемпионом для «среднего по году» остаётся честно-проверенная
система E (~74%, прошла кросс-проверку на 2 источниках). Старый V2 65.9%
сохранён как база сравнения.

### 1. МЕХАНИКА СДЕЛКИ (основа всего)

Мы торгуем **бинарные опционы**, НЕ классический форекс:

- Инструмент: **EUR/USD**
- Экспирация: **5 часов** с момента входа
- Вход: **немедленно по текущей рыночной цене** (нет отложенных ордеров)
- Нет TP/SL. Результат определяется ТОЛЬКО ценой в момент закрытия:
  - Прогноз BUY → плюс, если цена закрытия **выше** цены входа (хотя бы на 1 тик)
  - Прогноз SELL → плюс, если цена закрытия **ниже** цены входа

**Почему 5 часов:** выбрано сознательно, чтобы отфильтровать шум младших таймфреймов. Если прогноз правильный, у движения есть запас времени проявить себя (2–3 часа), и остаётся резерв.

### 3. ЯДРО СИСТЕМЫ: ИЕРАРХИЯ АНАЛИЗА (обновлено 12.06.2026 по решению JONY)

**ОСНОВА системы — КРУПНЫЕ ИГРОКИ.** График — это их след: один трейдер не двигает структуру, её двигают банки и фонды. Мы читаем, где они прячутся, что хотят и КУДА поведут цену — и идём ЗА ними. Порядок весов (сверху — главное):

**⚡ С 12.06.2026 направление выбирает ПРАВИЛО V2** (выведено из честного бэктеста 365 дней / 1197 сделок, полный отчёт — `BACKTEST_365D.md`): 54.3% против 49.8% у старой иерархии. Приоритет: (1) цена у низа 20-барного диапазона → BUY (61%); (2) RSI H1 ≤25/≥75 → контртренд (56%); (3) крупные игроки с сильным вердиктом → за ними; (4) тренд — только на откате, вдогонку запрещено (вдогонку = 44%); (5) умеренный bias крупных / EMA. Иерархия ниже остаётся базой анализа и объяснений.

1. **КРУПНЫЕ ИГРОКИ** (модуль `smart_money.py`, см. `SMART_MONEY.md`) — структура рынка (BOS/CHoCH), дисплейсмент, ордер-блоки, имбалансы (FVG), пулы ликвидности, премиум/дискаунт, свипы. При сильном вердикте (|балл| ≥ 3.5) — именно ОНИ задают направление прогноза, даже против EMA-тренда (тогда цель ближе: 1.0×ATR).
2. **Технический анализ** — RSI, ATR, положение в диапазоне, качество входа 🟢/🟡/🔴.
3. **Тренд** — EMA50/200 на D1/H4/H1 (если крупные не дали сильного вердикта, направление берётся отсюда).
4. **Фундаментал** — ЕЦБ/ФРС, ставки, экономика.
5. **Новости** — жёсткий фильтр: в окне сделки красная новость → прогноз НЕ выдаётся.
6. **Макроэкономика** — общий фон (риск-он/риск-офф).
7. **Прочие факторы** — сессии (см. §6a), сезонность и т.д.

**Железные правила:** против свежего съёма ликвидности не входим никогда. Во флэте без вердикта крупных не входим. История 11.06.2026 (урок JONY): съём вчерашнего лоу + CHoCH вверх = крупные покупают → нужно было BUY, хотя EMA-тренд показывал вниз. Теперь система так и делает.

### 4. АЛГОРИТМ АВТОСКАНЕРА — ПОШАГОВО (точная логика `scanner.py`)

Автосканер работает **каждые 5 часов** (cron `0 */5 * * *` UTC — слоты 00/05/10/15/20) и **КАЖДЫЙ запуск присылает прогноз** в Telegram-бот **FOREX AI 2026** (решение JONY 12.06.2026: минимум 3 прогноза в день обязательно; каждое присланное сообщение = вход СРАЗУ, никаких «не заходить сейчас»). Исключение одно — красная новость в окне 5ч: тогда вместо прогноза приходит новостной брифинг, вход — после новости. К каждому прогнозу прилагается оценка качества (A+/A/B) и список «что против нас» — чтобы JONY сам выбирал размер риска.

Условия сигнала: момент входа 🟢 (или 🟡 при вердикте крупных + confluence), минимум 2 независимые группы подтверждений из 5 (крупные игроки / тренд / confluence+ / свип в нашу сторону / RSI-эдж), нет блокеров (флэт, свип против, 3+ confluence-фактора против, красная новость). После сигнала — пауза 5ч (сделка идёт). Если сетапа нет — раз в ~5ч короткий статус «сканирую, момента нет». Доп. модуль `confluence_plus.py` (COT CFTC, DXY, US10Y, RSI-экстремум, круглые уровни, свечи) — см. `WINRATE_PLAN.md`.

#### Шаг 1. Данные (Yahoo Finance, символ `EURUSD=X`)
- M15 — 5 дней истории
- H1 — 60 дней
- D1 — 2 года (недельная и месячная история читается отсюда)
- H4 — ресемпл из H1

#### Шаг 2. Определение тренда (мультитаймфрейм, EMA50/EMA200)
Для каждого ТФ: `up` если цена > EMA200 и EMA50 ≥ EMA200; `down` — зеркально; иначе `flat`.

```
score = 2.0×D1 + 1.5×H4 + 1.0×H1        (up=+1, down=−1, flat=0; диапазон −4.5…+4.5)
score > +0.3  → BUY (по тренду)
score < −0.3  → SELL (по тренду)
иначе         → флэт: направление по наклону EMA50(H1), метка «нет тренда»
```
Сила: |score| ≥ 2.5 — сильный; ≥ 0.8 — умеренный; меньше — флэт.

#### Шаг 3. Качество ВХОДА 🟢/🟡/🔴 (вход на откате, не вдогонку)
`pos` = положение цены в диапазоне последних 20 баров H1 (0 = у лоёв, 1 = у хаёв), + RSI(14) M15:

- **SELL:** pos ≥ 0.6 или RSI_M15 ≥ 55 → 🟢 (цена на отскоке вверх — продавать по тренду); pos ≥ 0.4 → 🟡; иначе 🔴 (цена уже внизу — не гнаться)
- **BUY:** pos ≤ 0.4 или RSI_M15 ≤ 45 → 🟢 (откат вниз — покупать по тренду); pos ≤ 0.6 → 🟡; иначе 🔴
- Флэт → всегда 🔴 «пропустить»

#### Шаг 3a. КРУПНЫЕ ИГРОКИ — основа (модуль `smart_money.py`, добавлено 12.06.2026)
Перед выбором направления сканер собирает «вердикт крупных» из 7 компонентов (каждый голосует −2…+2, подробно в `SMART_MONEY.md`):
структура H4/H1 (HH/HL vs LH/LL, BOS/CHoCH) · дисплейсмент (свеча с телом ≥1.6×ATR) · ордер-блоки · имбалансы FVG · пулы ликвидности (равные хаи/лоу = КУДА поведут цену; ближайший пул может стать ЦЕЛЬЮ сделки) · премиум/дискаунт диапазона H4 · свежие свипы.

```
|балл| ≥ 3.5  → крупные ЗАДАЮТ направление (даже против EMA-тренда; цель ближе, 1.0×ATR)
балл ≥ +2 / ≤ −2 → умеренный вердикт: совпал с трендом → уверенность +2..+3, против → −3 и 🟢→🟡
иначе          → крупные не определились, направление по тренду (Шаг 2)
```
В каждом прогнозе блок «🏦 КРУПНЫЕ ИГРОКИ» + полное чтение графика их глазами в «Разборе простыми словами».

#### Шаг 4. Ловушки ликвидности / крупные игроки (модуль `liquidity_traps`)
Логика smart money: стопы толпы стоят за хаем/лоем прошлой недели и вчерашнего дня — туда ходят крупные игроки снимать ликвидность.

- Уровни: **хай/лоу прошлой НЕДЕЛИ** (вес 2 — лучший по бэктесту) и **вчерашний хай/лоу** (вес 1)
- Свежий съём = за последние 12 баров H1 цена проколола уровень и закрылась обратно (ложный пробой)
- **Съём ПРОТИВ нашего направления → принудительно 🔴 «ПРОПУСТИТЬ», уверенность −3** (никогда не входить против свежего съёма)
- Съём В НАШУ сторону (например, ложный пробой недельного хая при SELL) → уверенность +2 (единственный паттерн с реальным эджем ~55%)
- Уровни недели/дня печатаются в каждом прогнозе (строка 🪤)

#### Шаг 5. НОВОСТНОЙ ФИЛЬТР — жёсткая блокировка прогноза (обновлено 11.06.2026)

**Главное правило: если в 5-часовом окне сделки есть невышедшая важная новость — ПРОГНОЗ НЕ ВЫДАЁТСЯ ВООБЩЕ.** Исход новости заранее неизвестен, и она ломает любой техсетап — в такие окна не входим.

Источник: ForexFactory `ff_calendar_thisweek.json`, события **High impact** по **USD и EUR**.

**Сканер блокирует прогноз, если:**
- новость выходит в ближайшие 5 часов (попадает в окно сделки), ИЛИ
- новость уже вышла, но рынок ещё не успокоился (не прошёл «буфер переваривания»)

**Буферы после выхода новости (когда снова безопасно):**
| Тип события | Буфер |
|---|---|
| Решение по ставке (ФРС/ЕЦБ), FOMC, пресс-конференция | 2 часа |
| NFP, CPI | 1.5 часа |
| Выступления / показания (Speaks, Testifies) | 1 час |
| Остальные данные (PPI, PMI, Retail Sales, GDP…) | 1 час |

**Вместо прогноза бот присылает разбор:** по каждой новости — что это (объяснение простым языком), кто выступает (например, Пауэлл/Лагард), время выхода в UTC+5, прогноз/предыдущее значение, и главное — **точное время, после которого безопасно заходить** (= время последней новости + её буфер). Следующий чистый прогноз приходит автоматически.

**Уже вышедшие новости тоже учитываются:**
- вышла < буфера назад → блокировка (рынок ещё трясёт)
- вышла > буфера, но < 3 часов назад → прогноз выдаётся, но с предупреждением «рынок может ещё отыгрывать» и уверенность −2
- календарь недоступен (после 3 попыток + кэш до 12ч) → прогноз с жёстким предупреждением «проверь forexfactory.com сам, при красных новостях НЕ заходи», уверенность −2

Глубокий разбор политики/макро (тон ФРС/ЕЦБ, геополитика) — по запросу у ИИ; сканер ловит календарные события.

#### Шаг 6. Уверенность (честная, без накрутки)
```
conf = 50
+4 сильный тренд / +2 умеренный / 0 флэт
+3 вход 🟢 / +1 🟡 / −2 🔴
+2 ловушка за нас / −3 ловушка против нас
−3 скорая важная новость
итог зажимается в коридор 48–58%
```
Коридор 48–58% — принципиально: больше система честно дать не может (см. раздел 2).

- Инвалидация: противоположная граница 20-барового диапазона H1 (или ±1.5×ATR, что дальше)

### 5. ПРАВИЛА ИСПОЛНЕНИЯ (дисциплина = главный эдж)

1. **Входить только на 🟢.** 🟡 — можно с уменьшенным размером. **🔴 — всегда пропуск**, без исключений. Это и есть защита от «разворота через 3–4 часа»: не входить вдогонку и не входить во флэте.
2. **Риск ≤ 1–2% депозита на сделку.** Никогда не повышать ставку после проигрыша (никакого мартингейла).
3. **Никогда не входить против свежего съёма ликвидности** (см. шаг 4).
4. **Никогда не входить, пока в 5-часовом окне есть невышедшая важная новость.** Сканер это делает автоматически (блокирует прогноз и присылает время безопасного входа), но правило действует и при ручных входах: красная новость USD/EUR впереди = пропуск.
5. **Вести журнал сделок** (дата, направление, цена входа, метка 🟢/🟡/🔴, уверенность, результат). Без журнала из 100+ сделок невозможно понять, работает ли система. ЭТО ЕЩЁ НЕ ВНЕДРЕНО — следующий обязательный шаг.
6. Оценивать систему минимум на 100 сделках. Серия из 3–5 побед/поражений ничего не значит.
7. Не менять стратегию на ходу. Любое изменение — только после бэктеста на истории.

Как это реализовано на самом деле:

1. **Цифры (свечи M15/H1/H4/D1)** сканер берёт из Yahoo Finance chart API. Важно понимать:
   у TradingView НЕТ публичного API для выгрузки свечей — он сам лишь ПОКАЗЫВАЕТ котировки
   брокеров/банков. Цены EUR/USD у Yahoo и TradingView — один и тот же межбанковский поток,
   расхождение долей пипса. То есть **сканер считает по тем же свечам, которые видны на TradingView**.
2. **Глазами сканер смотрит именно на TradingView:** при каждом прогнозе открывается реальный
   браузер, грузится живой график TradingView `FX:EURUSD` на **M15, H1, H4 и D1**, делаются
   скриншоты — они и приходят в Telegram вместе с прогнозом. Если TradingView недоступен,
   строятся свои свечные графики (fallback), о чём честно видно по виду картинок.
3. Парсить сами свечи ИЗ TradingView скрейпингом мы сознательно НЕ делаем: это нарушает их
   правила, ломается при каждом обновлении сайта и ДОБАВЛЯЕТ ошибок, а не точности. Точность
   даёт не «другая картинка», а правильная математика на правильных свечах — что и сделано.

### `features_v2.py` — Расчёт признаков (только прошлое, без заглядывания в будущее)

```python
"""Build a per-(pair, hour-bar) features + 3h-outcome table. No lookahead:
all features come from bars <= t; outcome = close[i+3] vs close[i]."""
import pandas as pd, numpy as np, os

PAIRS14 = ["EURUSD","GBPUSD","USDJPY","AUDUSD","USDCHF","USDCAD","NZDUSD","EURJPY",
           "EURGBP","GBPJPY","AUDJPY","EURCHF","CHFJPY","CADJPY"]

def rsi(s, n=14):
    d = s.diff(); up = d.clip(lower=0).rolling(n).mean(); dn = (-d.clip(upper=0)).rolling(n).mean()
    return 100 - 100/(1 + up/dn.replace(0, np.nan))

def ema(s, n): return s.ewm(span=n, adjust=False).mean()

def prep(sym):
    h = pd.read_pickle(f"data/{sym}.pkl").copy()
    c = h["close"]
    h["rsi"] = rsi(c)
    h["hi20"] = h["high"].rolling(20).max(); h["lo20"] = h["low"].rolling(20).min()
    h["pos"] = (c - h["lo20"]) / (h["hi20"] - h["lo20"])
    tr = pd.concat([h["high"]-h["low"], (h["high"]-c.shift()).abs(), (h["low"]-c.shift()).abs()], axis=1).max(axis=1)
    h["atr"] = tr.rolling(14).mean()
    h["atrp"] = h["atr"].rolling(500).rank(pct=True)        # vol regime percentile
    h["atr_rel"] = h["atr"] / h["atr"].rolling(200).median() # rel to median (golden zone 1.1-1.4)
    h["dist"] = (c - ema(c, 50)) / h["atr"]
    h["ret5"] = c.pct_change(5)
    h["ret24"] = c.pct_change(24)
    h["e200sl"] = ema(c, 200).diff(20)
    h["h4tr"] = ema(c, 200).diff(4)
    # 24h sweep: previous 24h high/low taken out then price back inside
    hi24 = h["high"].shift(1).rolling(24).max(); lo24 = h["low"].shift(1).rolling(24).min()
    h["sweep_lo"] = ((h["low"] < lo24) & (c > lo24)).astype(int)
    h["sweep_hi"] = ((h["high"] > hi24) & (c < hi24)).astype(int)
    # candle body direction of last bar
    h["body"] = (c - h["open"]) / h["atr"]
    h["sym"] = sym
    return h

def build(horizon=3):
    rows = []
    for sym in PAIRS14:
        h = prep(sym)
        c = h["close"].values
        n = len(h)
        fut = np.full(n, np.nan)
        fut[:n-horizon] = c[horizon:] - c[:n-horizon]
        h = h.assign(move=fut)
        h = h.iloc[550:]                       # warmup for rolling windows
        h = h[h["move"].notna() & (h["move"] != 0)]
        h = h[h["time"].dt.weekday < 5]
        rows.append(h)
    df = pd.concat(rows, ignore_index=True)
    df["hour"] = df["time"].dt.hour
    df["dow"] = df["time"].dt.weekday
    df["month"] = df["time"].dt.strftime("%Y-%m")
    return df

if __name__ == "__main__":
    df = build(3)
    df.to_pickle("candidates_h3.pkl")
    print(len(df), "rows", df["time"].min(), "->", df["time"].max())
    print(df.groupby("sym").size())
```

### `news_calendar.py` — Календарь красных новостей (фильтр новостей)

```python
"""Approximate red-news calendar (UTC) for Jun 2025 - Jun 2026.
NFP: first Friday 12:30 UTC (summer DST) / 13:30 (winter).
CPI: typical release days 12:30/13:30 UTC (approx mid-month).
FOMC: 19:00 UTC winter / 18:00 summer. ECB: 13:15 UTC press conf 13:45.
"""
import pandas as pd

def first_friday(y, m):
    d = pd.Timestamp(year=y, month=m, day=1)
    return d + pd.Timedelta(days=(4 - d.weekday()) % 7)

def is_dst(ts):  # US DST approx: 2nd Sunday Mar - 1st Sunday Nov
    return ts.month in (4,5,6,7,8,9,10) or (ts.month==3 and ts.day>=9) or (ts.month==11 and ts.day<=1)

FOMC = ["2025-06-18","2025-07-30","2025-09-17","2025-10-29","2025-12-10",
        "2026-01-28","2026-03-18","2026-04-29","2026-06-17"]
ECB  = ["2025-06-05","2025-07-24","2025-09-11","2025-10-30","2025-12-18",
        "2026-02-05","2026-03-19","2026-04-30","2026-06-11"]
CPI_DAYS = {  # approx US CPI release dates
    "2025-06":11,"2025-07":15,"2025-08":12,"2025-09":11,"2025-10":15,"2025-11":13,"2025-12":10,
    "2026-01":13,"2026-02":11,"2026-03":11,"2026-04":10,"2026-05":12,"2026-06":10}

def red_events(start="2025-06-01", end="2026-06-30"):
    evs = []
    rng = pd.period_range(start[:7], end[:7], freq="M")
    for p in rng:
        y, m = p.year, p.month
        ff = first_friday(y, m)
        evs.append(ff + pd.Timedelta(hours=12 if is_dst(ff) else 13, minutes=30))     # NFP
        cd = CPI_DAYS.get(f"{y}-{m:02d}")
        if cd:
            ts = pd.Timestamp(year=y, month=m, day=cd)
            evs.append(ts + pd.Timedelta(hours=12 if is_dst(ts) else 13, minutes=30)) # CPI
    for d in FOMC:
        ts = pd.Timestamp(d); evs.append(ts + pd.Timedelta(hours=18 if is_dst(ts) else 19))
    for d in ECB:
        ts = pd.Timestamp(d); evs.append(ts + pd.Timedelta(hours=13, minutes=15))
    return sorted(pd.Timestamp(e, tz="UTC") for e in evs)

if __name__ == "__main__":
    evs = red_events()
    print(len(evs), "events"); [print(e) for e in evs[:8]]
```

```python
def run(label, seeds, params=None, save=None):

        hours = set(slots) | {s+1 for s in slots}
        test = DF[(DF["month"]==m) & DF["hour"].isin(hours) & (DF["time"]>=start) & (DF["time"]<=end)].copy()
        train = DF[DF["month"] < m]
        if len(train) < 5000 or test.empty: continue
        X = train[FEATS].astype(float); y = train["up"]
        ps = []
        for sd in seeds:
            clf = HistGradientBoostingClassifier(random_state=sd, **params)
            clf.fit(X, y)
            ps.append(clf.predict_proba(test[FEATS].astype(float))[:,1])
        p = np.mean(ps, axis=0)
        test["p"] = p; test["conf"] = np.abs(p-0.5)
        test["dir"] = np.where(test["p"]>0.5, 1, -1)
        test["win"] = (test["move"]>0) == (test["dir"]>0)
        for day in test["time"].dt.date.unique():
            g_day = test[test["time"].dt.date == day]
            for s in slots:
                t_norm = g_day[g_day["hour"]==s]
                if t_norm.empty: continue
                use = t_norm
                if news_hit(t_norm["time"].iloc[0]):
                    sh = g_day[g_day["hour"]==s+1]
                    if not sh.empty: use = sh
                r = use.nlargest(1, "conf").iloc[0]
                all_trades.append({"t": r["time"], "sym": r["sym"], "win": bool(r["win"]), "month": r["month"]})
    tr = pd.DataFrame(all_trades)
    mo = tr.groupby("month")["win"].agg(["mean","size"])
    print(f"{label}: {100*tr['win'].mean():.1f}% n={len(tr)} мес≥65%: {int((mo['mean']>=0.65).sum())}/{len(mo)} худший: {100*mo['mean'].min():.1f}%", flush=True)
    if save: tr.to_pickle(save)

if __name__ == "__main__":
    run("V6a 7seed+news", seeds=range(7), save="mlv6a.pkl")
    run("V6b deep 7seed+news", seeds=range(7),
        params=dict(max_iter=250, max_depth=6, learning_rate=0.05, min_samples_leaf=120), save="mlv6b.pkl")
    run("V6c shallow 7seed+news", seeds=range(7),
        params=dict(max_iter=120, max_depth=3, learning_rate=0.08, min_samples_leaf=300), save="mlv6c.pkl")
```
