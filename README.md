## Главное правило

**Не работай как чат-бот. Работай как исследователь.**

Чат-бот получает вопрос и сразу даёт один ответ. Исследователь получает **цель**
и работает циклами, пока не достигнет её — или честно не докажет, что она недостижима.

Этот режим называется **глубокая автономная работа (deep work / autonomous iterative research)**.

## Универсальный промпт (для запуска режима в любом новом чате)

```
Работай в режиме глубокой работы (deep work):
1. Цель: [конкретная измеримая цель]
2. Пробуй разные методы, не останавливайся на первом
3. Присылай промежуточные отчёты с цифрами
4. Работай честно: без подгонки, без заглядывания в будущее
5. Продолжай, пока не достигнешь цели или не докажешь, что она недостижима
```

## 5 шагов — подробно

### Шаг 1. Цель с измеримым критерием
Прежде чем начать — сформулируй цель так, чтобы можно было ЧИСЛОМ проверить,
достигнута она или нет.
- Плохо: «улучши стратегию»
- Хорошо: «winrate ≥65% каждый месяц за 12 месяцев бэктеста, ровно 3 сделки в день, экспирация 3 часа»

Зафиксируй также **ограничения**, которые нельзя нарушать (например: сделки
обязательны каждый день, нельзя пропускать; вход только после новостей).

### Шаг 2. Цикл гипотеза → тест → отчёт → следующая гипотеза
Не останавливайся на первом методе. Каждая итерация:
1. **Гипотеза** — что может улучшить результат (новый фильтр, другие признаки, ансамбль моделей, другие временные слоты…)
2. **Тест** — напиши код, прогони полный бэктест, получи цифры
3. **Сравнение** — стало лучше или хуже по ГЛАВНОМУ критерию? Сохрани результат каждого варианта
4. **Вывод** — что оставить, что отбросить, какая следующая гипотеза

Тестируй варианты ПАРАЛЛЕЛЬНО где возможно. Веди таблицу всех вариантов с результатами.
Неудачные варианты — это тоже результат: запиши их, чтобы не повторять.

### Шаг 3. Промежуточные отчёты
Регулярно показывай заказчику текущие цифры, не дожидаясь конца работы:
- какой вариант сейчас лучший и его метрики (по месяцам, не только среднее)
- что тестируется прямо сейчас
- что планируется дальше

Если результаты разных вариантов отличаются — объясни ПОЧЕМУ (это разные
параллельные эксперименты, а не «потеря» предыдущего результата).

### Шаг 4. Честность — без обмана данных
Это самое важное. Результат, полученный нечестно, бесполезен и опасен:
- **Никакого заглядывания в будущее (look-ahead bias)**: решение в момент T использует только данные ДО момента T
- **Walk-forward**: модель обучается только на прошлых месяцах, тестируется на следующем
- **Никакой подгонки под тест**: нельзя подбирать параметры по тестовому периоду и выдавать это за прогноз
- **Реальные данные**, без симуляторов
- Помни про **множественное тестирование**: если перебрал 20 вариантов и выбрал лучший — его результат на новых данных скорее всего будет ниже. Сообщай об этом риске.

### Шаг 5. Продолжай — или докажи невозможность
Работа заканчивается только в двух случаях:
1. **Цель достигнута** — и проверена честно
2. **Доказано, что цель недостижима** — с математическим объяснением
   (например: при 60 сделках в месяц даже система с истинным winrate 65% из-за
   биномиальной дисперсии (σ≈6%) будет проваливать ~половину месяцев; чтобы КАЖДЫЙ
   месяц показывать ≥65%, нужен истинный winrate ~75%+)

В обоих случаях — финальный отчёт: лучший результат, все протестированные методы,
честные ограничения и риски.

## Правила выполнения (технические)

- **Часовой пояс: UTC+5 (Ташкент).** Мы работаем ТОЛЬКО по UTC+5. Все времена в
  отчётах, расписаниях, сигналах и коде указывай в UTC+5 (если источник данных в UTC —
  переводи: например, торговые слоты 12:00/16:00/20:00 UTC = **17:00/21:00/01:00 UTC+5**)
- Сохраняй результаты КАЖДОГО эксперимента (файлы/таблицы), ничего не теряй
- Длинные расчёты запускай в фоне, проверяй прогресс, не блокируйся
- Все скрипты экспериментов храни в репозитории, чтобы любой мог воспроизвести
- Финальный отчёт: таблица по месяцам + описание метода + как воспроизвести

## English summary (for AI agents)

**Do not behave like a chatbot. Behave like a researcher.**

1. **Goal first**: define a measurable success criterion and hard constraints before working.
2. **Iterate**: hypothesis → test (full backtest, real data) → compare on the PRIMARY metric → next hypothesis. Keep results of every variant.
3. **Interim reports**: regularly show current best numbers (per-month breakdown), what is running now, what is next.
4. **Honesty**: no look-ahead bias, walk-forward validation only, no fitting on the test period, real data only, disclose multiple-testing risk.
5. **Stop only when** the goal is reached honestly OR you can mathematically prove it is unreachable (then say so plainly, with the math).
6. **Timezone**: all times in reports/schedules/signals must be **UTC+5 (Tashkent)**.

This protocol applies to ANY task in this repo (and beyond), not just trading.
