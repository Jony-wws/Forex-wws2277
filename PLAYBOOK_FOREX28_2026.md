# Часть I — Фундамент

## Глава 1. Бинарные опционы 5 часов: математика выживания

### 1.1. Природа инструмента
Бинарный опцион — это бинарная ставка: цена через 5 часов будет ВЫШЕ или НИЖЕ страйка.
- **Rise** (Call) — цена выше страйка на момент экспирации.
- **Fall** (Put) — цена ниже страйка на момент экспирации.

В отличие от спот-форекса, нет стоп-лосса и тейк-профита: либо payout, либо потеря всей ставки. Это меняет всю математику.

### 1.2. Payout 70% — точка безубыточности
- Ставка `S`. Win → получаем `S × 1.70` (то есть прибыль `+0.70 × S`). Loss → теряем `S` (–1.00 × S).
- Безубыточная вероятность: `p × 0.70 = (1 − p) × 1.00` → **p = 1.00 / 1.70 = 58.82%**.
- Чтобы расти, нужно стабильно `p ≥ 60%`. Целевой нижний порог в playbook = **70%** для запаса.
- При p=70%, payout=70%: матожидание сделки = `0.70 × 0.70 − 0.30 × 1.00 = +0.19 × S` (+19%).
- При p=80%: EV = `0.80 × 0.70 − 0.20 × 1.00 = +0.36 × S` (+36%).

### 1.3. Kelly Criterion для бинарного опциона
Классическая формула Kelly: `f* = (bp − q) / b`, где
- `b` = odds = payout/loss = 0.70/1.00 = **0.70**
- `p` = вероятность выигрыша
- `q` = 1 − p

Примеры оптимальной ставки:
| p | f* (Kelly) | f*/2 (Half-Kelly, рекомендуется) |
|---|---|---|
| 60% | 17.1% | 8.6% |
| 65% | 25.0% | 12.5% |
| 70% | 32.9% | **16.4%** |
| 75% | 40.7% | 20.4% |
| 80% | 48.6% | 24.3% |

**Важно.** Полный Kelly — слишком агрессивно (большая просадка). На бинарных опционах используем **Half-Kelly** или **Quarter-Kelly**:
- При нашем целевом p=70% → **5–7% от депозита** = соответствует Quarter-Kelly с большим запасом безопасности.
- **Жёсткий ограничитель: max 20%** (см. правило Jony) — никогда не превышать, даже если вероятность кажется выше.

### 1.4. Серии проигрышей (drawdown protection)
Даже при p=70% серия из 4 подряд лоссов имеет вероятность `(0.30)^4 = 0.81%` → раз в ~123 сделок (примерно раз в 1–2 месяца при 1–3 сигналах/день).
- При ставке 5%: после 4 лоссов депозит = 0.95^4 = **81.45%** от исходного.
- При ставке 10%: после 4 лоссов = 0.90^4 = **65.61%** — больно.
- При ставке 20%: после 4 лоссов = 0.80^4 = **40.96%** — катастрофа.

**Правило:** при p~70%, ставка ≤ 5–7% от депозита.

### 1.5. Экспоненциальный рост vs линейный
- Линейный (фиксированная ставка $X): депозит растёт медленно, не использует капитализацию.
- Экспоненциальный (% от депозита): первые сделки маленькие, но через 50–100 win-сделок размер ставки растёт сам.
- **Используем экспоненциальный.** Пересчитываем размер ставки после каждой сделки.

EV рассчитан при p=70%, 60 сделок/мес = 3 сделки/день × 20 рабочих дней. При 1–2 сделках/день EV пропорционально меньше, но и риск меньше.

### 3.4. Stop-loss на дневной/недельный уровень
- **Дневной stop:** 3 проигрыша подряд → стоп на сегодня (психология).
- **Недельный stop:** −15% от понедельничного депозита → стоп на неделю.
- **Месячный stop:** −25% → пауза, ревью системы (что сломалось?).

### 3.5. Не использовать «доливку после лосса» (мартингейл)
**Запрещено.** Это путь к нулю. Размер следующей сделки = % от текущего депозита, не привязан к результату предыдущей.

## Глава 5. Currency Strength Meter (CSM) — формула и источники

### 5.1. Базовая формула (классическая, single-window)
Для каждой из 8 валют:
```
strength_X = average over all 7 pairs containing X of (% change in favor of X over period T)
```
Период `T`: D1 (1 день), W1 (5 дней), M1 (20 дней).

Пример для USD на D1:
- Пары: EURUSD, GBPUSD, AUDUSD, NZDUSD, USDJPY, USDCHF, USDCAD
- Если EURUSD упал на 0.5% → USD вырос на 0.5% против EUR.
- Если USDJPY вырос на 0.3% → USD вырос на 0.3% против JPY.
- Сумма всех 7 направленных изменений / 7 = strength_USD.

### 5.2. Усиленная формула (multi-window blended)
Слабость одного дня — шум. Используем взвешенное среднее:
```
strength_blended_X = 0.50 × strength_D1 + 0.30 × strength_W1 + 0.20 × strength_M1
                   + macro_tilt_X (нормированный hawkish_rank/8 × 0.20%)
```
Это даёт устойчивый ранкинг, нечувствительный к одному дню flow.

### 5.3. Алгоритмический ранкинг 1–8
- Сортируем 8 валют по `strength_blended_X` от слабейшей (#1) к сильнейшей (#8).
- Минимальный gap для входа: **|rank_base − rank_quote| ≥ 4**.

## Глава 6. Risk-On / Risk-Off матрица

### 6.1. Что такое Risk-On / Risk-Off (RoRo)
Глобальный режим аппетита к риску:
- **Risk-On:** инвесторы покупают рискованные активы (акции, EM, AUD, NZD, GBP). Продают safe-havens (USD, JPY, CHF, gold).
- **Risk-Off:** инвесторы бегут в safe-havens. Продают рискованные активы.

### 6.2. Прокси-индикаторы
| Индикатор | Risk-On значение | Risk-Off значение |
|---|---|---|
| **DXY** (US Dollar Index) | падает ↓ | растёт ↑ |
| **SPX** (S&P 500) | растёт ↑ | падает ↓ |
| **VIX** (volatility index) | <20 | >25 |
| **Gold (XAU/USD)** | падает или флэт | растёт ↑ |
| **JPY** | падает ↓ | растёт ↑ |
| **CHF** | флэт/падает | растёт ↑ |
| **AUD/JPY** (классический RoRo proxy) | растёт ↑ | падает ↓ |
| **US 10Y yield** | растёт ↑ | падает ↓ (flight to bonds) |
| **Oil (WTI)** | растёт (рост экономики) | сложно |

# Часть III — Технический Confluence (multi-timeframe)

## Глава 8. D1 тренд — старший таймфрейм

### 8.1. Структура (Higher-High / Higher-Low)
- **Бычий тренд:** последовательность HH (higher highs) и HL (higher lows). Каждый pullback держится выше предыдущего минимума.
- **Медвежий тренд:** последовательность LH (lower highs) и LL (lower lows).
- **Боковик/range:** highs и lows примерно на одном уровне, нет ясной структуры.

### 8.2. 200 EMA на D1
- Цена выше 200 EMA → долгосрочный bullish bias.
- Цена ниже 200 EMA → долгосрочный bearish bias.
- Цена ON 200 EMA → нейтрально, осторожно (часто отбойный уровень).

### 8.3. ADX(14) на D1
- ADX > 25 → есть тренд на дневке.
- ADX 20–25 → слабый тренд.
- ADX < 20 → флэт, не торгуем по тренду.

### 8.4. EMA stack 8/21/55 на D1 (опционально)
- Все три EMA выровнены восходяще (8>21>55) → bullish.
- Все три выровнены нисходяще (8<21<55) → bearish.
- Перепутаны → индикация смены тренда.

### 8.5. D1 trend label
| Цена vs 200 EMA | Структура | EMA stack | ADX | D1 trend |
|---|---|---|---|---|
| > | HH/HL | 8>21>55 | >25 | **BUY** |
| < | LH/LL | 8<21<55 | >25 | **SELL** |
| любое | смешанное | смешанное | <25 | **FLAT** |

## Глава 9. H4 momentum — основа сетапа

### 9.1. Зачем H4
H4 — компромисс между D1 (медленный) и H1 (шумный). H4 свечи показывают institutional flow.

### 9.2. EMA stack 8/21/55 на H4
Тот же принцип, что и D1, но более чувствительный.
- 8>21>55 + цена выше всех → **BUY**
- 8<21<55 + цена ниже всех → **SELL**

### 9.3. ADX(14) на H4 — главный фильтр тренда
- **ADX > 25** = настоящий тренд → ✅ можно торговать
- ADX 20–25 = слабеющий или зарождающийся тренд → ⚠️ ждём подтверждения
- **ADX < 20** = флэт → ❌ не торгуем
- При ADX > 30 — сильный тренд, повышенная вероятность.
- ADX > 50 — экстремально сильный, но риск разворота (после долгого тренда).

**+DI vs −DI:**
- +DI > −DI → бычий моментум
- −DI > +DI → медвежий

### 9.4. RSI(14) на H4
- RSI > 70 в bullish trend → не overbought, продолжение
- RSI < 30 в bearish trend → не oversold, продолжение
- Дивергенция RSI (см. Глава 24) — сигнал ослабления

### 9.5. MACD на H4
- MACD line > signal line + histogram > 0 → bullish момент
- MACD < signal + hist < 0 → bearish момент
- MACD crossover в направлении тренда D1 = подтверждение входа

## Глава 10. H1 entry trigger

### 10.1. Не основа сетапа, а триггер
H1 не определяет тренд — D1+H4 определяют. H1 нужен для ТОЧКИ ВХОДА.

### 10.2. Pin bar / отбойная свеча
- Длинный фитиль в нужном направлении (отбой от уровня)
- Тело маленькое, фитиль 60%+ свечи
- Закрытие в направлении сделки

### 10.3. Engulfing на H1
- Bullish engulfing на support зоне → BUY триггер
- Bearish engulfing на resistance зоне → SELL триггер

### 10.4. Break-and-retest
1. Цена пробивает ключевой уровень (Глава 8 — D1 swing high/low)
2. Возвращается ретестировать пробитый уровень
3. Отбивается → вход

### 10.5. Не входим на H1 без D1+H4 confluence
H1 trigger без D1+H4 поддержки — fake-out. Жёсткое правило.

## Глава 11. Объём, ATR, ADR, Bollinger, Keltner

### 11.1. ATR(14) — Average True Range
Средняя величина движения за период (включая gaps).
- Используем для оценки волатильности и установки целей.
- ATR на H4 актуальной пары = «в среднем 4h-свеча даёт X пипсов».

### 11.2. ADR — Average Daily Range
Средний дневной диапазон за 20 дней:
```
ADR = mean(high_i - low_i) for i in last 20 days
```

### 11.3. ADR-фильтр (критически важно для бинарных опционов 5h)
Цель: цена должна закрыться **далеко** от барьера, а не на волоске.

| ADR использовано (% за сегодня) | Действие |
|---|---|
| 0–40% | ✅ Много запаса для движения |
| 40–60% | ⚠️ Запас уменьшается, осторожно |
| 60–80% | ⚠️ Граница, проверь дополнительные сигналы |
| **>80%** | ❌ **Не входим** — цена уже прошла дневной диапазон, реверс или истощение |
| >100% | ❌❌ Особо рискованно — день экстремальной волатильности |

### 11.4. Bollinger Bands (20, 2σ)
- Цена касается верхней BB → overbought (часто)
- Цена касается нижней BB → oversold
- BB squeeze (узкий канал) → ожидание прорыва, не входим до него
- BB expansion → сильный тренд

### 11.5. Keltner Channels (20, 2 ATR)
Аналог BB, но на ATR. Прорыв Keltner = подтверждение тренда.

### 11.6. Volume (если доступен)
На spot форексе настоящего объёма нет, есть «tick volume» (количество тиков).
- Растущий объём в направлении тренда = подтверждение
- Объёмный спайк на пробое = ICT liquidity sweep

# Часть IV — Smart Money / ICT (institutional flow)

> Источник: ICT (Inner Circle Trader, Michael J. Huddleston). Также Wyckoff, и SMC framework. Эти методы видят, ГДЕ институционалы строят позиции и ГДЕ забирают liquidity.

## Глава 12. Структура рынка: BOS, CHoCH, MSS

### 12.1. Bullish vs Bearish структура
- **Bullish:** последовательность HH/HL. Каждый pullback держится выше предыдущего HL.
- **Bearish:** последовательность LH/LL.

### 12.2. BOS (Break of Structure)
- В bullish тренде: цена пробивает предыдущий HH → продолжение тренда.
- В bearish: цена пробивает предыдущий LL → продолжение.
- BOS = подтверждение текущего тренда.

### 12.3. CHoCH (Change of Character)
- В bullish тренде: цена пробивает предыдущий HL ВНИЗ → возможный разворот.
- В bearish: цена пробивает предыдущий LH ВВЕРХ → возможный разворот.
- CHoCH = первый признак смены тренда.

### 12.4. MSS (Market Structure Shift)
Подтверждённый CHoCH с импульсным движением и закрытием за уровнем — официальная смена тренда.

### 12.5. Применение в playbook
- **D1 BOS в bullish:** усиливает BUY-сетап на пары.
- **D1 CHoCH:** жёлтый флаг — система переходит в осторожный режим.
- **D1 MSS:** серьёзный сигнал смены, не торгуем по старому тренду 1–2 дня.

## Глава 13. Order Blocks (OB)

### 13.1. Определение
**Order Block** — последняя противоположная свеча перед сильным импульсом.
- **Bullish OB:** последняя bearish свеча перед сильным bullish импульсом. Институционалы там купили.
- **Bearish OB:** последняя bullish свеча перед сильным bearish импульсом. Институционалы там продали.

### 13.2. Сильный OB vs слабый
**Сильный OB:**
- Сформировался ПОСЛЕ liquidity sweep
- В discount zone (нижняя половина диапазона) для bullish
- В premium zone (верхняя половина) для bearish
- Сопровождается impulse-баром (range ≥ 3× median)
- На H4 или D1 — приоритетнее

**Слабый OB:** случайная свеча без контекста, быстро ломается.

### 13.3. Как использовать в 5h binary
- Идентифицируем OB на H4/D1 по текущему направлению тренда.
- Когда цена возвращается в зону OB → высокая вероятность отбоя.
- Вход в направлении OB при confluence с другими фильтрами.

### 13.4. Невалидация OB
Если цена пробила и закрылась за OB на H4 → OB больше не работает. Игнорируем.

## Глава 14. Fair Value Gaps (FVG) и imbalances

### 14.1. Определение
**FVG** — гэп между свечами, видимый как пустое пространство:
- **Bullish FVG:** high свечи 1 ниже low свечи 3 (свеча 2 — мощный импульс), цена пропустила средний диапазон.
- **Bearish FVG:** low свечи 1 выше high свечи 3.

### 14.2. Логика «магнита цены»
Рынок стремится «закрыть» FVG. На крипторынке fill rate ~70–80%, на форексе ~65–75%.
- Незаполненный FVG = магнит для цены.

### 14.3. Применение
- D1 bullish FVG ниже текущей цены → если цена пойдёт вниз, это таргет (или зона входа для BUY).
- D1 bearish FVG выше → таргет для SELL.
- H4 FVG — entry zone в направлении старшего тренда.

### 14.4. Чем шире FVG, тем мощнее
Узкий FVG (1–2 пипса на H4) — шум. FVG 10–15 пипсов на H4 = серьёзная имбалансная зона.

## Глава 15. Liquidity Sweeps (стопы и стоп-хант)

### 15.1. Где сидит ликвидность
- Над равными хаями (equal highs) — buy-side liquidity (BSL): шорт-стопы и buy-стопы breakout.
- Под равными лоями (equal lows) — sell-side liquidity (SSL).
- На round numbers (1.10, 1.20) — психологические уровни.
- На сессионных хай/лоу (Asia high/low — особенно).
- На daily high/low — самый частый таргет.

### 15.2. Liquidity sweep / stop hunt
- Цена резко выносит уровень (на 5–15 пипсов)
- Собирает стопы / breakout-ордера
- Разворачивается обратно

**Это НЕ пробой**, это институциональный сбор ликвидности.

### 15.3. Asian range raid
В начале London (07:00–09:00 UTC = 12:00–14:00 Душанбе) часто:
1. Sweeping Asian high (если bearish setup)
2. Затем true направление в обратную сторону
3. Это и есть **Judas swing** (см. Глава 16)

### 15.4. Применение
- При sweeping daily high → ждём отбой → вход SELL (если confluence).
- При sweeping daily low → BUY.
- Объёмный спайк на пробое = подтверждение sweep.

### 15.5. Невалидация sweep
Если цена закрылась H4-свечой ЗА уровнем (не ретест, реальный пробой) → это не sweep, это true breakout. Меняем тренд.

## Глава 16. Judas Swing и Kill Zones

### 16.1. Judas swing
Ложное движение в первые 30–90 минут после открытия London/NY:
- Часто противоположно true направлению дня
- Заманивает retail в неверную сторону
- Затем разворот — настоящий тренд

### 16.2. London Kill Zone
- **07:00–10:00 UTC = 12:00–15:00 Душанбе**
- Самые большие движения дня
- Часто Judas в первые 30–60 мин, затем true direction

### 16.3. NY Kill Zone
- **13:00–16:00 UTC = 18:00–21:00 Душанбе**
- Overlap с London → пик волатильности
- US data releases чаще всего в этом окне

### 16.4. Asian range
- **23:00–06:00 UTC = 04:00–11:00 Душанбе**
- Узкий диапазон, used as liquidity для London raid

### 16.5. Защита от Judas
- **Не входим в первые 30 мин после открытия London/NY**
- Если уже в позиции и видим Judas против себя — НЕ удваиваемся, ждём confluence для true direction
- Лучший вход — ПОСЛЕ Judas в направлении настоящего тренда

## Глава 17. Optimal Trade Entry (OTE) и Premium/Discount

### 17.1. Discount vs Premium
В диапазоне (от swing low до swing high):
- **Discount zone:** нижние 50% диапазона. Здесь покупаем (по bullish тренду).
- **Premium zone:** верхние 50%. Здесь продаём (по bearish тренду).

### 17.2. OTE = Fibonacci 62%–79%
ICT OTE — pullback ровно в зону fib 0.62–0.79 от последнего impulse:
- В bullish: цена идёт от swing low (0%) до swing high (100%), pullback к 62–79% = OTE для BUY.
- В bearish: от swing high (0%) до swing low (100%), pullback к 62–79% = OTE для SELL.

### 17.3. Confluence OTE
Самые сильные сетапы — где OTE ПЕРЕСЕКАЕТСЯ с:
- D1 Order Block
- H4 FVG
- Свинг-уровнем

### 17.4. Stop / target для бинарного
В 5h binary нет SL/TP, но логика:
- Вход = OTE 70%
- Невалидация = пробой 100% уровня
- Цель = противоположный край диапазона (или дальше)

# Часть V — Свечной анализ

> Источник: классические японские свечи + статистика 2025–2026 (MQL5, FXNX, BrokerRank backtests).
>
> **Главный принцип:** одиночная свеча НИКОГДА не сигнал в вакууме. Свеча работает только в правильном **контексте** (тренд, уровень, объём, confluence).

## Глава 18. Одиночные свечи

### 18.1. Doji — нерешительность
- Тело крошечное (open ≈ close)
- Длинные фитили в обе стороны
- **Win rate: ~42%** (overrated в одиночку, негативная expectancy)
- **Контекст обязателен:** на support/resistance, после длинного тренда — может быть разворот. В range — шум.
- Для нас: doji + RSI divergence + level = сигнал ослабления тренда.

### 18.2. Hammer / Hanging Man
- Маленькое тело наверху, длинный нижний фитиль (60%+ свечи)
- На support после downtrend = bullish reversal (Hammer)
- На resistance после uptrend = bearish reversal (Hanging Man)
- **Win rate: ~62%** (Hammer на H4 в forex)
- Risk-reward 1:2.5, expectancy +0.42R

### 18.3. Shooting Star / Inverted Hammer
- Маленькое тело внизу, длинный верхний фитиль
- На resistance после uptrend = bearish reversal (Shooting Star)
- На support после downtrend = bullish (Inverted Hammer, слабее)
- **Win rate: ~55%** (Shooting Star)

### 18.4. Pin Bar
- Объединяющий термин для Hammer/Shooting Star
- Длинный фитиль 66%+, тело маленькое в противоположной стороне
- На уровне (S/R, EMA, fib) → reversal
- **Pin bar + level + EMA = классический ICT/SMC trigger**

### 18.5. Marubozu
- Свеча без фитилей (или почти), большое тело
- Сильный momentum в направлении тела
- **Bullish Marubozu** на пробое = подтверждение BUY
- **Bearish Marubozu** на пробое = SELL

### 18.6. Spinning Top
- Маленькое тело, оба фитиля среднего размера
- Нерешительность, может предвестить разворот
- Слабее Doji в плане сигнала

## Глава 19. Двухсвечные паттерны

### 19.1. Bullish Engulfing
- Свеча 1: bearish (small/medium)
- Свеча 2: bullish, тело которой ПОЛНОСТЬЮ перекрывает тело свечи 1
- На support после downtrend = strong reversal
- **Win rate: ~58%, R:R 1:3, expectancy +0.48R**

### 19.2. Bearish Engulfing
- Зеркально. На resistance после uptrend.
- Аналогичный win rate.

### 19.3. Tweezer Top / Bottom
- Две свечи с почти одинаковыми high (Top) или low (Bottom)
- На уровне S/R = двойной отбой
- Win rate ~55% (требует подтверждения)

### 19.4. Harami (Inside Bar)
- Свеча 2 ПОЛНОСТЬЮ внутри тела свечи 1
- Слабее Engulfing — часто пауза, не разворот
- **Win rate ~50–52%** (overrated)
- Для нас: Inside Bar = сжатие → ждём breakout direction.

### 19.5. Piercing Line / Dark Cloud Cover
- Piercing Line: bullish reversal — закрытие 2-й свечи >50% тела 1-й (bearish)
- Dark Cloud Cover: bearish reversal — закрытие 2-й <50% тела 1-й (bullish)
- Слабее Engulfing, но подтверждает разворот

## Глава 20. Трёхсвечные паттерны

### 20.1. Morning Star (bullish reversal)
- Свеча 1: large bearish
- Свеча 2: small body (Doji или Spinning Top), gap down
- Свеча 3: large bullish, закрытие выше середины 1-й
- На support после downtrend = strong reversal
- **Win rate: ~55–60%** (3-свечные обычно сильнее одно-/двухсвечных)

### 20.2. Evening Star (bearish reversal)
- Зеркально.

### 20.3. Three White Soldiers (продолжение bullish)
- Три bullish свечи подряд, каждая закрывает выше предыдущей
- Минимальные верхние фитили
- На пробое из консолидации = strong continuation

### 20.4. Three Black Crows (продолжение bearish)
- Зеркально.

### 20.5. Three Inside Up / Down
- Inside bar + breakout свеча в направлении breakout
- Ранний вход в направлении пробоя

## Глава 21. Контекст важнее паттерна

### 21.1. Правила контекста
1. Паттерн должен формироваться **на уровне** (S/R, EMA, fib OTE, OB, FVG).
2. Тренд старшего TF должен **поддерживать** направление паттерна.
3. **Объём** в направлении паттерна = подтверждение.
4. **Ретест** уровня + паттерн = высокая вероятность.

### 21.2. Когда паттерн НЕ работает
- В range (без чёткого тренда)
- На малых TF (M1, M5) без подтверждения старшего
- В одиночку, без других confluence

### 21.3. Pattern + Filter формула
```
Качество сигнала = Pattern_strength + Level_strength + Trend_alignment + Volume_confirmation
```
Минимум 3/4 этих компонентов в плюс — иначе пропуск.

# Часть VI — Антиловушки (избегание ложных моментов)

## Глава 22. Все 9 ловушек: детекция и защита

### 22.1. Judas Swing
- **Что:** ложное движение в первые 30–90 мин London/NY
- **Детекция:** резкий импульс против ожидаемого тренда + быстрый разворот
- **Защита:** не входим в первые 30 мин после открытия London (07:00 UTC) и NY (13:00 UTC). Жди разворота → вход в true direction.

### 22.2. Bull Trap
- **Что:** Цена пробивает resistance → retail покупает → цена падает
- **Детекция:** 
  - Пробой на низком объёме
  - RSI bearish divergence
  - Нет ретеста + закрытия H4 за уровнем
  - Цена в premium zone D1
- **Защита:** ждём H4 close ЗА resistance + retest + bullish trigger

### 22.3. Bear Trap
- **Что:** Цена пробивает support → retail продаёт → цена растёт
- **Детекция:** аналогично, зеркально
- **Защита:** ждём H4 close ниже support + retest + bearish trigger

### 22.4. News Spike Trap
- **Что:** На новости цена летит 100+ пипсов → разворот
- **Детекция:** новостной всплеск, экстремальная волатильность, ширина >2× ATR
- **Защита:** **30 мин ДО + 60 мин ПОСЛЕ** любой High Impact новости — НЕ ВХОДИМ.

### 22.5. Whipsaw / Pila
- **Что:** Резкие движения вверх-вниз, ловит обе стороны
- **Когда:** Низкая ликвидность (Asia без AUD/JPY catalysts), перед важной новостью, в Asian range
- **Детекция:** ADX <20, BB squeeze, цена в узком диапазоне
- **Защита:** Не входим при ADX <20.

### 22.6. Liquidity Grab
- **Что:** Цена выносит equal highs/lows → собирает стопы → разворот
- **Детекция:**
  - Цена приближается к видимому S/R уровню
  - Резкий вынос на 5–15 пипсов с быстрым возвратом
  - Объёмный спайк на пике
- **Защита:** На очевидных уровнях — жди sweep → вход ПОСЛЕ. Не до.

### 22.7. Fake Breakout
- **Что:** Цена выходит за уровень на 5–15 пипсов → возвращается
- **Детекция:** на H1/H4 свеча НЕ закрылась за уровнем
- **Защита:** ждём H4 close за уровнем + retest + согласие фундамента

### 22.8. Stop Hunting
- **Что:** Институционалы двигают цену к скоплению стопов, забирают, разворачивают
- **Детекция:** цена идёт точно к round number или sw high/low, sharp wick
- **Защита:** в бинарных опционах нет SL → нам выгодно. Но мы не входим против stop hunt — ждём, пока он отработает.

### 22.9. Consolidation Trap
- **Что:** Рынок сужается → иллюзия нейтральности → взрывное движение
- **Детекция:** BB squeeze, ATR падает, объём низкий
- **Защита:** В консолидации НЕ входим. Ждём прорыв + ретест.

## Глава 23. News blackout — детальные правила

### 23.1. Tier 1 (High Impact) — обязательный blackout
- **NFP** (Non-Farm Payroll) — 1-я пятница месяца, 13:30 UTC
- **CPI** (US, EU, UK, JP) — раз в месяц
- **Central bank rates** (FOMC, ECB, BoE, BoJ, RBA, RBNZ, BoC, SNB)
- **GDP** (квартальный)
- **PMI Flash** (месячный)

**Правило:** **30 мин ДО + 60 мин ПОСЛЕ** любого Tier-1 события по любой из двух валют пары — НЕ ВХОДИМ.

### 23.2. Tier 2 (Medium Impact) — осторожно
- Retail Sales, Industrial Production, Trade Balance, Consumer Confidence, ADP
- 15 мин ДО + 30 мин ПОСЛЕ — осторожно, лучше пропуск.

### 23.3. Tier 3 (Low Impact) — игнор
- Foreign Reserves, Eco Watchers, etc. Не влияют.

### 23.4. Источники календаря
- **ForexFactory.com** — главный, фильтр по impact
- **Investing.com** — backup
- **Tradingeconomics.com** — overview

### 23.5. Окно сделки 5 часов и календарь
**Перед входом:** проверяем все события на пару следующие 5 часов.
- Если какое-либо Tier-1 событие попадает в окно → пропуск ИЛИ вход с уменьшенной ставкой ИЛИ ждём после события.
- Если только Tier-2 → разрешено, но снижаем ставку.

## Глава 24. Дивергенции (RSI, MACD, OBV)

### 24.1. Regular Bearish Divergence
- Цена: higher highs (HH)
- Индикатор: lower highs (LH)
- → Bullish momentum слабеет → готовим SELL
- **RSI div daily — win rate ~68%** (highest among all divergence types per backtested 500 S&P stocks)

### 24.2. Regular Bullish Divergence
- Цена: lower lows (LL)
- Индикатор: higher lows (HL)
- → готовим BUY

### 24.3. Hidden Bullish Divergence
- Цена: higher lows (HL)
- Индикатор: lower lows (LL)
- → продолжение uptrend (continuation)

### 24.4. Hidden Bearish Divergence
- Цена: lower highs (LH)
- Индикатор: higher highs (HH)
- → продолжение downtrend

### 24.5. MACD vs RSI
- RSI div лучше работает на dailies
- MACD div лучше на H4 / D1, histogram divergence сильнее signal-line divergence (+12% per backtest)

### 24.6. Multi-indicator divergence
**RSI + MACD + OBV** одновременно подтверждающие — точность **~78%** (но встречается редко, ~8–12% trading opportunities).

### 24.7. False divergence — 25–35% всех сигналов
**Подтверждение обязательно:**
- Свечной разворотный паттерн
- Пробой структуры (CHoCH)
- Объёмный спайк

## Глава 25. Объёмное подтверждение и фигуры

### 25.1. Volume на форексе
Spot форекс не имеет cleared volume (OTC market). Используем:
- **Tick volume** (количество тиков от брокера) — proxy
- **CME Futures volume** (для major пар через CME mini-futures)

### 25.2. Volume + breakout
- Прорыв на **растущем** объёме = подтверждение
- Прорыв на **низком** объёме = подозрение на fake breakout
- Импульсная свеча с volume spike = institutional flow

### 25.3. OBV (On-Balance Volume)
- OBV растёт + цена растёт → trend continuation
- OBV падает + цена растёт → bearish divergence (готовим SELL)

### 25.4. Volume profile (POC, VAH, VAL)
- POC (Point of Control) = price с наибольшим объёмом в session/day
- VAH/VAL = upper/lower bound 70% volume area
- Цена возвращается к POC → отбой или пробой
- **POC = магнит для цены**, как FVG

## Глава 26. Чеклист 8 фильтров confluence

### 26.1. Полный чеклист
| # | Фильтр | Условие | Источник |
|---|---|---|---|
| 1 | **Сила валют** | Gap rank ≥ 4 | CSM (Глава 5) |
| 2 | **D1 trend** | Совпадает с направлением (BUY/SELL/FLAT) | Глава 8 |
| 3 | **H4 trend** | Совпадает с D1 | Глава 9 |
| 4 | **ADX H4** | > 25 | Глава 9 |
| 5 | **EMA stack H4** | 8>21>55 (BUY) или 8<21<55 (SELL) | Глава 9 |
| 6 | **Сессия** | Лондон, Overlap, или допустимая Asia (только AUD/NZD/JPY) | Глава 2 |
| 7 | **News + traps** | Чисто на 5 часов | Глава 23, 22 |
| 8 | **Фундамент** | Поддерживает направление (RoRo + макро + новости ЦБ) | Глава 6, 7 |

### 26.2. Решение по баллам
- **8/8 = ИДЕАЛЬНЫЙ сигнал**, вероятность 80–88%
- **7/8 = СИЛЬНЫЙ сигнал**, вероятность 70–78%
- 6/8 = СЛАБЫЙ → ⚠️ можно только при бонусных факторах (см. ниже)
- 5/8 = ⛔ ПРОПУСК
- < 5/8 = ⛔⛔ полный пропуск

### 26.3. Бонусные факторы (повышают вероятность)
- ICT OB + FVG в OTE (Глава 17) → +5%
- Wyckoff Spring/Upthrust на старшем TF → +3%
- COT report показывает институционалов в направлении → +3%
- Multi-indicator divergence (RSI + MACD) → +5%
- Volume spike на пробое в нашем направлении → +3%

### 26.4. Penalty факторы (снижают вероятность)
- Цена в premium для BUY (или discount для SELL) — против OTE → −5%
- ADR использовано >60% → −3% за каждые 10%
- Близко к Tier-2 news → −5%
- Friday после 17:00 UTC → −3% (week-end gap)
- Признаки неопределённости (Iran headlines в день анализа) → −5%

## Глава 27. Формула вероятности (4 компонента)

### 27.1. Базовая формула
```
P_base = 50 + 5 × confluence_count  
       (где confluence_count = количество "✓" из 8 фильтров)
```
- 5/8 → P_base = 75% (но gate 7+ требуется → 5/8 = пропуск)
- 7/8 → P_base = 85%
- 8/8 → P_base = 90%

### 27.2. Корректировки
```
P = P_base + ADX_strength_bonus + ADR_buffer + macro_tilt + bonuses − penalties
```
- **ADX_strength_bonus:**
  - ADX 25–30 → +0
  - ADX 30–40 → +2
  - ADX 40–50 → +4
  - ADX >50 → +5 (но риск разворота, ограничиваем +5)
- **ADR_buffer:**
  - Использовано <40% → +3
  - 40–60% → 0
  - 60–80% → −3
  - >80% → пропуск (правило)
- **macro_tilt** (см. Глава 4 ranking):
  - rank gap 4 → 0
  - rank gap 5 → +1
  - rank gap 6 → +2
  - rank gap 7 → +3

### 27.3. Cap и пол
- **Probability cap = 92%** (никогда не показываем 100%)
- **Probability floor for entry = 70%**
- Между 70 и 92% — это рабочий диапазон.

### 27.4. Пример расчёта
**Пример 1: BUY AUD/CHF (идеальный сетап в Лондон overlap)**
- 8/8 confluence = 90% base
- ADX 35 = +2
- ADR 30% used = +3
- Rank gap 7 (AUD #1 hawkish vs CHF #8 dovish) = +3
- ICT OB confluence = +5
- = 103% → cap **92%**

**Пример 2: SELL EUR/NZD (слабый Asia setup)**
- 6/8 confluence = 80% base
- ADX 29 = +0
- ADR 41% used = +0
- Rank gap 3 = слишком мало (gap minimum 4) → дисквалификация

## Глава 29. Playbook 5h сделки (pre/entry/monitor/close)

### 29.1. Pre-trade (15 мин до окна)
1. Открыть `MACRO_TABLE.md` — проверить актуальность (если >5 дней — обновить).
2. Открыть FF calendar — проверить новости на 6 часов вперёд.
3. Запустить `python3 forex_analysis/strength_28.py` — получить ranking.
4. Открыть TradingView — посмотреть top 3 кандидатов на D1/H4/H1.
5. Применить чеклист 8 фильтров (Глава 26).
6. Рассчитать вероятность (Глава 27).
7. Принять решение по дереву (Глава 28).

### 29.2. Entry
1. Записать в журнал: пара, направление, время входа, страйк, ставка $, вероятность %, причины.
2. Открыть позицию.
3. Снять скриншот D1, H4, H1 с TradingView.
4. Сохранить скриншот в `HISTORY/<date>/<trade-id>/`.

### 29.3. Monitoring (3 чек-пойнта)
- **+1 час:** проверить, идёт ли движение в нашу сторону. Если резко против — **не закрываем** (бинарное всё равно), но записываем в журнал.
- **+3 часа:** аналогично. Видим ли мы Wyckoff phase change? Liquidity sweep? OB-уровень удержался?
- **+4:30 часа** (за 30 мин до экспирации): финальная оценка. Записать прогноз исхода.

### 29.4. Close (после 5h)
1. Записать результат: WIN / LOSS, прибыль/убыток, % движения.
2. **Mandatory review** (даже на WIN): что прошло по плану? что было неожиданно?
3. На LOSS — определить тип ловушки (если была), записать урок.

### 29.5. После сделки
1. Не входить вторую сделку немедленно. Минимум **1 час паузы** после закрытия (психология).
2. Максимум **3 сделки за календарный день**.
3. Если 2 LOSS подряд — **дневной стоп**, не торгуем до завтра.

## C. Расчётные формулы (cheat sheet)

### C.1. ATR(14)
```
TR_i = max(high_i - low_i, |high_i - close_{i-1}|, |low_i - close_{i-1}|)
ATR_i = (ATR_{i-1} * 13 + TR_i) / 14   (Wilder smoothing)
```

### C.2. ADR(20)
```
ADR = mean(high_i - low_i)  for last 20 daily bars
```

### C.3. ADX(14)
```
+DM_i = max(high_i - high_{i-1}, 0)  if (high_i - high_{i-1}) > (low_{i-1} - low_i) else 0
-DM_i = max(low_{i-1} - low_i, 0)  if (low_{i-1} - low_i) > (high_i - high_{i-1}) else 0
+DI = 100 * EMA(+DM, 14) / ATR(14)
-DI = 100 * EMA(-DM, 14) / ATR(14)
DX = 100 * |+DI - -DI| / (+DI + -DI)
ADX = EMA(DX, 14)
```

### C.4. EMA
```
EMA_i = α * price_i + (1 - α) * EMA_{i-1}
α = 2 / (n + 1)
```

### C.5. RSI(14)
```
gain_i = max(close_i - close_{i-1}, 0)
loss_i = max(close_{i-1} - close_i, 0)
avg_gain = EMA(gain, 14)   (Wilder)
avg_loss = EMA(loss, 14)
RS = avg_gain / avg_loss
RSI = 100 - 100 / (1 + RS)
```

### C.6. MACD(12, 26, 9)
```
MACD_line = EMA(close, 12) - EMA(close, 26)
Signal_line = EMA(MACD_line, 9)
Histogram = MACD_line - Signal_line
```

### C.7. Currency Strength (single window)
```
For each currency C:
  S_C = mean over 7 pairs containing C of (% change in C's favor)
ranking = sort(currencies by S_C, ascending)
rank[C] = index + 1  (weakest = 1, strongest = 8)
```

## E. Wyckoff Schema (краткое)

### Accumulation phases (A → E)
- **Phase A:** stop downtrend (PS = preliminary support, SC = selling climax, AR = automatic rally, ST = secondary test)
- **Phase B:** building cause, range
- **Phase C:** **Spring** — false breakdown, забирает стопы
- **Phase D:** SOS (sign of strength), LPS (last point of support)
- **Phase E:** markup phase — trend up

### Distribution phases (зеркально)
- Phase A: stop uptrend (PSY, BC, AR, ST)
- Phase B: range
- Phase C: **Upthrust** — false breakout
- Phase D: SOW (sign of weakness)
- Phase E: markdown — trend down

### Применение
- На D1 пары находим в какой phase. Если в Phase E (markup или markdown) — confluence сильный для тренда.
- Spring/Upthrust на H4 — высоковероятностный entry trigger.
