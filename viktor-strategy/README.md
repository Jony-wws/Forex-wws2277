## 1. МЕХАНИКА СДЕЛКИ (основа всего)

Мы торгуем **бинарные опционы**, НЕ классический форекс:

- Инструмент: **EUR/USD**
- Экспирация: **5 часов** с момента входа
- Вход: **немедленно по текущей рыночной цене** (нет отложенных ордеров)
- Нет TP/SL. Результат определяется ТОЛЬКО ценой в момент закрытия:
  - Прогноз BUY → плюс, если цена закрытия **выше** цены входа (хотя бы на 1 тик)
  - Прогноз SELL → плюс, если цена закрытия **ниже** цены входа
- Размер движения НЕ важен — важно только направление на момент экспирации
- Выплата ~80% → **точка безубыточности = 55.6% винрейта**

**Следствие:** даже в боковике можно выиграть, но боковик ОПАСЕН (направление случайно). Поэтому система требует тренд и помечает флэт как «пропустить».

**Почему 5 часов:** выбрано сознательно, чтобы отфильтровать шум младших таймфреймов. Если прогноз правильный, у движения есть запас времени проявить себя (2–3 часа), и остаётся резерв.

## 2. ФИЛОСОФИЯ: ВЕРОЯТНОСТЬ, А НЕ ГАРАНТИЯ (зафиксировано, проверено бэктестами)

Это ключевая договорённость системы, к которой мы пришли через реальные бэктесты:

- Базовая вероятность угадать направление EUR/USD на 5ч ≈ **50%**
- **Гарантированных 70%+ в месяц НЕ СУЩЕСТВУЕТ** для направленной торговли FX на 5ч. Ни один набор индикаторов это не даёт.
- Честная цель системы: **55–58%** за счёт СЕЛЕКТИВНОСТИ (брать только лучшие моменты), а не за счёт добавления индикаторов.
- Улучшение системы = повышение избирательности входов, а НЕ усложнение анализа.
- Один выигрыш или малая серия = удача, не статистика. Оценивать только на 100+ сделках.

### Результаты РЕАЛЬНЫХ бэктестов (EURUSD=X, H1, ~17 000 баров, ~35 месяцев)

| Стратегия | Винрейт | Вывод |
|---|---|---|
| «PRO confluence» (мульти-индикаторная) | ~46.5% | хуже монетки |
| 5 факторов JONY (тренд+RSI+уровни+...) | ~49.5%, 0 из 35 месяцев ≥70% | подтверждено: 70%/мес недостижимо |

| Большинство SMC-«ловушек» на горизонте 5ч | ~50% (шум) | использовать только как фильтр |

Справка (почему дисциплина важнее «секретной» стратегии): 70–89% розничных трейдеров FX/CFD теряют деньги (данные ESMA), ~80% на бинарных опционах (ASIC). Типичные причины: овертрейдинг, нереальные цели, прыжки между стратегиями, отсутствие журнала сделок.

## 3. ЯДРО СИСТЕМЫ: ИЕРАРХИЯ АНАЛИЗА (обновлено 12.06.2026 по решению JONY)

**ОСНОВА системы — КРУПНЫЕ ИГРОКИ.** График — это их след: один трейдер не двигает структуру, её двигают банки и фонды. Мы читаем, где они прячутся, что хотят и КУДА поведут цену — и идём ЗА ними. Порядок весов (сверху — главное):

1. **КРУПНЫЕ ИГРОКИ** (модуль `smart_money.py`, см. `SMART_MONEY.md`) — структура рынка (BOS/CHoCH), дисплейсмент, ордер-блоки, имбалансы (FVG), пулы ликвидности, премиум/дискаунт, свипы. При сильном вердикте (|балл| ≥ 3.5) — именно ОНИ задают направление прогноза, даже против EMA-тренда (тогда цель ближе: 1.0×ATR).
2. **Технический анализ** — RSI, ATR, положение в диапазоне, качество входа 🟢/🟡/🔴.
3. **Тренд** — EMA50/200 на D1/H4/H1 (если крупные не дали сильного вердикта, направление берётся отсюда).
4. **Фундаментал** — ЕЦБ/ФРС, ставки, экономика.
5. **Новости** — жёсткий фильтр: в окне сделки красная новость → прогноз НЕ выдаётся.
6. **Макроэкономика** — общий фон (риск-он/риск-офф).
7. **Прочие факторы** — сессии (см. §6a), сезонность и т.д.

**Железные правила:** против свежего съёма ликвидности не входим никогда. Во флэте без вердикта крупных не входим. История 11.06.2026 (урок JONY): съём вчерашнего лоу + CHoCH вверх = крупные покупают → нужно было BUY, хотя EMA-тренд показывал вниз. Теперь система так и делает.

## 4. АЛГОРИТМ АВТОСКАНЕРА — ПОШАГОВО (точная логика `scanner.py`)

Автосканер сканирует рынок **каждые 30 минут** (cron `*/30 * * * *` UTC), но пишет в Telegram-бот **FOREX AI 2026** ТОЛЬКО когда сложился A+ сетап (режим «ТОЛЬКО ВХОД», введён 13.06.2026 по решению JONY: каждое присланное сообщение = вход СРАЗУ, никаких «не заходить сейчас»).

Условия сигнала: момент входа 🟢 (или 🟡 при вердикте крупных + confluence), минимум 2 независимые группы подтверждений из 5 (крупные игроки / тренд / confluence+ / свип в нашу сторону / RSI-эдж), нет блокеров (флэт, свип против, 3+ confluence-фактора против, красная новость). После сигнала — пауза 5ч (сделка идёт). Если сетапа нет — раз в ~5ч короткий статус «сканирую, момента нет». Доп. модуль `confluence_plus.py` (COT CFTC, DXY, US10Y, RSI-экстремум, круглые уровни, свечи) — см. `WINRATE_PLAN.md`.

### Шаг 1. Данные (Yahoo Finance, символ `EURUSD=X`)
- M15 — 5 дней истории
- H1 — 60 дней
- D1 — 2 года (недельная и месячная история читается отсюда)
- H4 — ресемпл из H1

### Шаг 2. Определение тренда (мультитаймфрейм, EMA50/EMA200)
Для каждого ТФ: `up` если цена > EMA200 и EMA50 ≥ EMA200; `down` — зеркально; иначе `flat`.

```
score = 2.0×D1 + 1.5×H4 + 1.0×H1        (up=+1, down=−1, flat=0; диапазон −4.5…+4.5)
score > +0.3  → BUY (по тренду)
score < −0.3  → SELL (по тренду)
иначе         → флэт: направление по наклону EMA50(H1), метка «нет тренда»
```
Сила: |score| ≥ 2.5 — сильный; ≥ 0.8 — умеренный; меньше — флэт.

### Шаг 3. Качество ВХОДА 🟢/🟡/🔴 (вход на откате, не вдогонку)
`pos` = положение цены в диапазоне последних 20 баров H1 (0 = у лоёв, 1 = у хаёв), + RSI(14) M15:

- **SELL:** pos ≥ 0.6 или RSI_M15 ≥ 55 → 🟢 (цена на отскоке вверх — продавать по тренду); pos ≥ 0.4 → 🟡; иначе 🔴 (цена уже внизу — не гнаться)
- **BUY:** pos ≤ 0.4 или RSI_M15 ≤ 45 → 🟢 (откат вниз — покупать по тренду); pos ≤ 0.6 → 🟡; иначе 🔴
- Флэт → всегда 🔴 «пропустить»

### Шаг 3a. КРУПНЫЕ ИГРОКИ — основа (модуль `smart_money.py`, добавлено 12.06.2026)
Перед выбором направления сканер собирает «вердикт крупных» из 7 компонентов (каждый голосует −2…+2, подробно в `SMART_MONEY.md`):
структура H4/H1 (HH/HL vs LH/LL, BOS/CHoCH) · дисплейсмент (свеча с телом ≥1.6×ATR) · ордер-блоки · имбалансы FVG · пулы ликвидности (равные хаи/лоу = КУДА поведут цену; ближайший пул может стать ЦЕЛЬЮ сделки) · премиум/дискаунт диапазона H4 · свежие свипы.

```
|балл| ≥ 3.5  → крупные ЗАДАЮТ направление (даже против EMA-тренда; цель ближе, 1.0×ATR)
балл ≥ +2 / ≤ −2 → умеренный вердикт: совпал с трендом → уверенность +2..+3, против → −3 и 🟢→🟡
иначе          → крупные не определились, направление по тренду (Шаг 2)
```
В каждом прогнозе блок «🏦 КРУПНЫЕ ИГРОКИ» + полное чтение графика их глазами в «Разборе простыми словами».

### Шаг 4. Ловушки ликвидности / крупные игроки (модуль `liquidity_traps`)
Логика smart money: стопы толпы стоят за хаем/лоем прошлой недели и вчерашнего дня — туда ходят крупные игроки снимать ликвидность.

- Уровни: **хай/лоу прошлой НЕДЕЛИ** (вес 2 — лучший по бэктесту) и **вчерашний хай/лоу** (вес 1)
- Свежий съём = за последние 12 баров H1 цена проколола уровень и закрылась обратно (ложный пробой)
- **Съём ПРОТИВ нашего направления → принудительно 🔴 «ПРОПУСТИТЬ», уверенность −3** (никогда не входить против свежего съёма)
- Съём В НАШУ сторону (например, ложный пробой недельного хая при SELL) → уверенность +2 (единственный паттерн с реальным эджем ~55%)
- Уровни недели/дня печатаются в каждом прогнозе (строка 🪤)

### Шаг 5. НОВОСТНОЙ ФИЛЬТР — жёсткая блокировка прогноза (обновлено 11.06.2026)

**Главное правило: если в 5-часовом окне сделки есть невышедшая важная новость — ПРОГНОЗ НЕ ВЫДАЁТСЯ ВООБЩЕ.** Исход новости заранее неизвестен, и она ломает любой техсетап — в такие окна не входим.

Источник: ForexFactory `ff_calendar_thisweek.json`, события **High impact** по **USD и EUR**.

**Сканер блокирует прогноз, если:**
- новость выходит в ближайшие 5 часов (попадает в окно сделки), ИЛИ
- новость уже вышла, но рынок ещё не успокоился (не прошёл «буфер переваривания»)

**Буферы после выхода новости (когда снова безопасно):**
| Тип события | Буфер |
|---|---|
| Решение по ставке (ФРС/ЕЦБ), FOMC, пресс-конференция | 2 часа |
| NFP, CPI | 1.5 часа |
| Выступления / показания (Speaks, Testifies) | 1 час |
| Остальные данные (PPI, PMI, Retail Sales, GDP…) | 1 час |

**Вместо прогноза бот присылает разбор:** по каждой новости — что это (объяснение простым языком), кто выступает (например, Пауэлл/Лагард), время выхода в UTC+5, прогноз/предыдущее значение, и главное — **точное время, после которого безопасно заходить** (= время последней новости + её буфер). Следующий чистый прогноз приходит автоматически.

**Уже вышедшие новости тоже учитываются:**
- вышла < буфера назад → блокировка (рынок ещё трясёт)
- вышла > буфера, но < 3 часов назад → прогноз выдаётся, но с предупреждением «рынок может ещё отыгрывать» и уверенность −2
- календарь недоступен (после 3 попыток + кэш до 12ч) → прогноз с жёстким предупреждением «проверь forexfactory.com сам, при красных новостях НЕ заходи», уверенность −2

Глубокий разбор политики/макро (тон ФРС/ЕЦБ, геополитика) — по запросу у ИИ; сканер ловит календарные события.

### Шаг 6. Уверенность (честная, без накрутки)
```
conf = 50
+4 сильный тренд / +2 умеренный / 0 флэт
+3 вход 🟢 / +1 🟡 / −2 🔴
+2 ловушка за нас / −3 ловушка против нас
−3 скорая важная новость
итог зажимается в коридор 48–58%
```
Коридор 48–58% — принципиально: больше система честно дать не может (см. раздел 2).

### Шаг 7. Цель и инвалидация (горизонт 5ч)
- Цель: цена ± 1.5×ATR(14, H1)
- Инвалидация: противоположная граница 20-барового диапазона H1 (или ±1.5×ATR, что дальше)

## 5. ПРАВИЛА ИСПОЛНЕНИЯ (дисциплина = главный эдж)

1. **Входить только на 🟢.** 🟡 — можно с уменьшенным размером. **🔴 — всегда пропуск**, без исключений. Это и есть защита от «разворота через 3–4 часа»: не входить вдогонку и не входить во флэте.
2. **Риск ≤ 1–2% депозита на сделку.** Никогда не повышать ставку после проигрыша (никакого мартингейла).
3. **Никогда не входить против свежего съёма ликвидности** (см. шаг 4).
4. **Никогда не входить, пока в 5-часовом окне есть невышедшая важная новость.** Сканер это делает автоматически (блокирует прогноз и присылает время безопасного входа), но правило действует и при ручных входах: красная новость USD/EUR впереди = пропуск.
5. **Вести журнал сделок** (дата, направление, цена входа, метка 🟢/🟡/🔴, уверенность, результат). Без журнала из 100+ сделок невозможно понять, работает ли система. ЭТО ЕЩЁ НЕ ВНЕДРЕНО — следующий обязательный шаг.
6. Оценивать систему минимум на 100 сделках. Серия из 3–5 побед/поражений ничего не значит.
7. Не менять стратегию на ходу. Любое изменение — только после бэктеста на истории.

## 6b. ОТКУДА СИСТЕМА БЕРЁТ ДАННЫЕ (честно, зафиксировано 11.06.2026)

**Требование JONY:** система должна смотреть на рынок «как на TradingView», таймфреймы M15/H1/H4/D1, без сторонних сервисов.

Как это реализовано на самом деле:

1. **Цифры (свечи M15/H1/H4/D1)** сканер берёт из Yahoo Finance chart API. Важно понимать:
   у TradingView НЕТ публичного API для выгрузки свечей — он сам лишь ПОКАЗЫВАЕТ котировки
   брокеров/банков. Цены EUR/USD у Yahoo и TradingView — один и тот же межбанковский поток,
   расхождение долей пипса. То есть **сканер считает по тем же свечам, которые видны на TradingView**.
2. **Глазами сканер смотрит именно на TradingView:** при каждом прогнозе открывается реальный
   браузер, грузится живой график TradingView `FX:EURUSD` на **M15, H1, H4 и D1**, делаются
   скриншоты — они и приходят в Telegram вместе с прогнозом. Если TradingView недоступен,
   строятся свои свечные графики (fallback), о чём честно видно по виду картинок.
3. Парсить сами свечи ИЗ TradingView скрейпингом мы сознательно НЕ делаем: это нарушает их
   правила, ломается при каждом обновлении сайта и ДОБАВЛЯЕТ ошибок, а не точности. Точность
   даёт не «другая картинка», а правильная математика на правильных свечах — что и сделано.
