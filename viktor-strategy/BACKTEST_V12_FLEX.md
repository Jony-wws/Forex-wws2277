# V12 «FLEX» — гибкая система без расписания (честный бэктест)

> Дата: 13.06.2026. Период бэктеста: 2025-06 … 2026-06 (13 месяцев, walk-forward).
> Модель: HistGradientBoosting seed-ensemble (champion V11 core), обучение только на месяцах < тестового (без подглядывания в будущее).

## Что это за система

В отличие от старой системы с фиксированными часами (12/16/20 UTC), **FLEX** работает так:

- **Нет расписания.** Система сканирует **28 пар × все часы** каждый день.
- Сама выбирает **3 самых уверенных входа в сутки** (порог уверенности = 80-й перцентиль по истории прошлых дней — причинно, без заглядывания вперёд).
- Вход = закрытие H1-бара (момент `t+1h`), **экспирация 3 часа** (выход `t+4h`).
- Мультитаймфрейм-признаки (M15 + H4 + D1).
- Новостной фильтр: если рядом красное событие (NFP/CPI/FOMC/ECB) — вход сдвигается на следующий час.
- **CADCHF удалён** (у брокера JONY этой пары нет).

## ⚠️ Важно: про разобранную ошибку с фидом (timezone)

Промежуточный прогон показывал на histdata всего 64.9% против 74.6% на Yahoo. Расследование выявило **баг часового пояса** в построении histdata-фида:

- histdata.com отдаёт время в **локальном времени Нью-Йорка (с переходом на летнее время)**, а пайплайн интерпретировал его как фиксированный EST.
- Из-за этого в DST-периоды свечи сдвигались на 1 час, и цены входа/выхода брались **не из той свечи** → ложное занижение win-rate.

**Доказательство (корреляция H1-доходностей Yahoo vs histdata):**

```
Старый (битый) фид:  lag 0 = 0.44,  lag -1 = 0.51   <- сигнал размазан по двум лагам
Исправленный фид:    lag 0 = 0.89,  lag -1/+1 ≈ 0    <- чёткое выравнивание
```

Фид пересобран с конвертацией `America/New_York` (DST-aware) → `data_m15_fixed`. После исправления независимый источник даёт 74.0%, совпадая с Yahoo. **Баг был только в проверочном слое, не в самой стратегии** — Yahoo-результат (74.3%) от него не зависел.

## ⚠️ Честные оговорки (реальные деньги — оптимизм запрещён)

1. **Это лучший из ~15 проверенных вариантов.** Лучший бэктест-результат почти всегда регрессирует вживую. Реалистичный live: **середина 60-х … низкие 70-е %**, а не гарантированные 74%.
2. **Сырой край модели тонкий: ~53.5%** точности на ВСЕХ сигналах подряд. Высокий win-rate берётся целиком за счёт **отбора только самых уверенных сигналов** (3/сутки), а не из «магии».
3. **«≥65% каждый месяц» — НЕ гарантия.** Слабые месяцы есть (независимый источник: ноябрь 63.2%, март 64.9%; Yahoo: неполный июнь 40% на 30 сделках). Просадочные периоды будут — нужен риск-менеджмент.
