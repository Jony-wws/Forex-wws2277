# 🤖 AI-обзор стратегии — авто-генерация

_Этот файл создаётся GitHub Actions workflow `ai_review.yml` каждый раз после завершения 5-часового цикла._

## 🧠 AI-обзор (GitHub Models · openai/gpt-4o-mini · бесплатно)

# Диагноз
1. **Низкий винрейт и стабильность**: Все три пары, которые показали сигналы, имеют винрейт ниже 60%, что не соответствует целям системы. Это указывает на недостаточную надежность текущих сигналов.
2. **Отсутствие сильных трендов**: В текущем цикле не было найдено ни одной валютной пары с явным трендом, что делает систему неэффективной для торговли. Пары показывают смешанные сигналы, что затрудняет принятие решений.
3. **Слабая предсказуемость**: Ожидаемые движения цен не соответствуют реальным результатам, что указывает на необходимость улучшения фильтров для более точного определения трендов.

# Рекомендации
1. **Увеличить STRONG_CONFIDENCE**: Установить значение на **92** (вместо 88) для повышения уверенности в сигналах.
2. **Увеличить STRONG_RATIO**: Установить значение на **0.65** (вместо 0.55) для повышения доли сильных сигналов среди всех сигналов.
3. **Увеличить STRONG_ADX_H1**: Установить значение на **30** (вместо 25) для фильтрации слабых трендов.
4. **Увеличить STRONG_ADX_H4**: Установить значение на **25** (вместо 20) для более строгого определения сильных трендов.
5. **Увеличить STRONG_PERSISTENCE**: Установить значение на **85** (вместо 80) для повышения требований к последовательности баров в сторону тренда.

# Что НЕ менять
- **Параметры, связанные с RSI и Aroon**: Эти индикаторы уже показывают хорошую предсказуемость и не требуют изменений.
- **Стратегии отката**: Они показывают определённую эффективность, и их не следует менять, пока не будет улучшена основная система фильтрации.

# Риски
1. **Снижение количества сигналов**: Ужесточение фильтров может привести к значительному уменьшению числа доступных сигналов, что может затруднить торговлю.
2. **Потенциальные упущенные возможности**: Более строгие параметры могут привести к пропуску прибыльных сделок, особенно в условиях волатильного рынка.
3. **Необходимость в адаптации**: Изменение порогов может потребовать дополнительного времени для тестирования и адаптации системы, что может временно снизить эффективность торговли.

- **WR за последние 5 часов:** —% (0 решений)
- **WR на 28-парном бэктесте:** —%

### Диагноз

- Пары в деградации (1): `EURNZD` — рассмотреть исключение из топа на 24ч.
