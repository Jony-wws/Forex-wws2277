# 🤖 AI-обзор стратегии — авто-генерация

_Этот файл создаётся GitHub Actions workflow `ai_review.yml` каждый раз после завершения 5-часового цикла._

## 🧠 AI-обзор (GitHub Models · openai/gpt-4o-mini · бесплатно)

# Диагноз
1. **Низкая стабильность сигналов**: В текущем цикле не удалось найти ни одной валютной пары с WR ≥ 70% на 5-дневном, 30-дневном и 365-дневном горизонте, что указывает на недостаточную надежность сигналов.
2. **Слабые тренды**: Большинство сигналов (особенно на коротких таймфреймах) не подтверждаются сильными трендами, что приводит к высокой вероятности ложных входов.
3. **Низкий винрейт**: Средний винрейт по всем парам не достигает целевого уровня, что указывает на необходимость ужесточения фильтров для повышения качества сигналов.

# Рекомендации
1. **Увеличить STRONG_CONFIDENCE**: Установить значение на уровне **92%** для повышения уверенности в сигналах.
2. **Увеличить STRONG_RATIO**: Установить значение на уровне **0.65** для повышения доли сильных сигналов среди всех сигналов.
3. **Увеличить STRONG_ADX_H1**: Установить значение на уровне **30** для фильтрации слабых трендов на часовом графике.
4. **Увеличить STRONG_ADX_H4**: Установить значение на уровне **25** для повышения требований к трендам на четырехчасовом графике.
5. **Увеличить STRONG_PERSISTENCE**: Установить значение на уровне **90%** (≥ 4 из 5 H1-баров в сторону тренда) для повышения устойчивости тренда.

# Что НЕ менять
- **STRONG_ADX_H4**: Параметр 20 может оставаться без изменений, так как он уже обеспечивает достаточный уровень фильтрации для четырехчасового графика.
- **Стратегии отката**: Оставить текущие стратегии, так как они могут быть полезны в условиях волатильности, если будут применены более строгие фильтры.

# Риски
1. **Снижение количества сигналов**: Ужесточение фильтров может привести к значительному сокращению числа доступных сигналов, что может снизить общую активность торговли.
2. **Пропуск потенциально прибыльных сделок**: Более строгие пороги могут привести к пропуску сделок, которые могли бы быть прибыльными, но не соответствуют новым критериям.
3. **Необходимость адаптации**: Трейдерам может потребоваться время для адаптации к новым параметрам и изменениям в стратегии, что может временно снизить эффективность торговли.

## 🧠 Эвристический AI-обзор (без LLM)

### Метрики
- **WR за последние 5 часов:** —% (0 решений)
- **WR на 28-парном бэктесте:** —%
- **Деградировавшие пары:** EURNZD

### Диагноз
- WR в норме — текущая жёсткость работает.
- Пары в деградации (1): `EURNZD` — рассмотреть исключение из топа на 24ч.

### Рекомендации к параметрам
- В `app/cycle.py._select_strict` добавить временный blacklist для деградировавших пар: ['EURNZD'].

### Точки правки
- `app/cycle.py`: `STRONG_CONFIDENCE`, `STRONG_RATIO`, `STRONG_ADX_H1`, `STRONG_ADX_H4`, `STRONG_PERSISTENCE`, `PREMIUM_ADX_H1`, `PREMIUM_PERSISTENCE`, `MIN_PICKS`, `MAX_PICKS`
- `scripts/cycle_5h.py`: `MIN_TRADES_PER_DAY`, `TOP_N`
