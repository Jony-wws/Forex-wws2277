# 🤖 AI-обзор стратегии — авто-генерация

_Этот файл создаётся GitHub Actions workflow `ai_review.yml` каждый раз после завершения 5-часового цикла._

## 🧠 AI-обзор (GitHub Models · openai/gpt-4o-mini · бесплатно)

# Диагноз
1. **Низкий винрейт**: В текущем цикле наблюдается низкий винрейт (52%) и отсутствие стабильных пар с WR ≥ 70%. Это указывает на недостаточную надежность сигналов.
2. **Недостаточная сила тренда**: Многие пары не показывают явного тренда, что затрудняет принятие решений. Например, пары с сигналами BUY и SELL имеют смешанные результаты и низкую уверенность.
3. **Проблемы с фильтрацией**: Текущие пороги фильтрации не обеспечивают достаточной строгости, что приводит к включению слабых сигналов и снижению общей эффективности.

# Рекомендации
1. **Увеличить STRONG_CONFIDENCE**: Установить значение на **92** (вместо 88) для повышения уверенности в сигналах.
2. **Увеличить STRONG_RATIO**: Установить значение на **0.65** (вместо 0.55) для повышения доли сильных сигналов среди всех.
3. **Увеличить STRONG_ADX_H1**: Установить значение на **30** (вместо 25) для фильтрации слабых трендов на часовом графике.
4. **Увеличить STRONG_ADX_H4**: Установить значение на **25** (вместо 20) для повышения требований к силе тренда на четырехчасовом графике.
5. **Увеличить STRONG_PERSISTENCE**: Установить значение на **85** (вместо 80) для повышения требований к последовательности тренда.

# Что НЕ менять
- **STRONG_ADX_H1 и H4**: Не менять параметры ADX для других таймфреймов, так как они уже обеспечивают необходимую фильтрацию.
- **Методология анализа**: Оставить текущую методологию анализа (мульти-ТФ, Aroon, моментум и т.д.), так как она уже показала свою эффективность в определенных условиях.

# Риски
1. **Снижение количества сигналов**: Ужесточение фильтров может привести к значительному снижению количества доступных сигналов, что может затруднить торговлю.
2. **Пропуск потенциально прибыльных сделок**: Более строгие пороги могут привести к пропуску сделок, которые могли бы быть прибыльными, но не соответствуют новым критериям.
3. **Необходимость адаптации**: Трейдерам может потребоваться время для адаптации к новым порогам и методам анализа, что может временно снизить их эффективность.

## 🧠 Эвристический AI-обзор (без LLM)

### Метрики
- **WR за последние 5 часов:** 52.0% (25 решений)
- **WR на 28-парном бэктесте:** —%
- **Деградировавшие пары:** AUDNZD

### Диагноз
- WR 52.0% — около break-even, есть запас для жёсткости.
- Пары в деградации (1): `AUDNZD` — рассмотреть исключение из топа на 24ч.

### Рекомендации к параметрам
- Поднять `STRONG_ADX_H1` 25 → 28, `STRONG_ADX_H4` 20 → 22.
- В `app/cycle.py._select_strict` добавить временный blacklist для деградировавших пар: ['AUDNZD'].

### Точки правки
- `app/cycle.py`: `STRONG_CONFIDENCE`, `STRONG_RATIO`, `STRONG_ADX_H1`, `STRONG_ADX_H4`, `STRONG_PERSISTENCE`, `PREMIUM_ADX_H1`, `PREMIUM_PERSISTENCE`, `MIN_PICKS`, `MAX_PICKS`
- `scripts/cycle_5h.py`: `MIN_TRADES_PER_DAY`, `TOP_N`
