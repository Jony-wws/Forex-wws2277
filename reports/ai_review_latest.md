# 🤖 AI-обзор стратегии — авто-генерация

_Этот файл создаётся GitHub Actions workflow `ai_review.yml` каждый раз после завершения 5-часового цикла._

## 🧠 AI-обзор (GitHub Models · openai/gpt-4o-mini · бесплатно)

# Диагноз
1. **Низкая стабильность сигналов**: В текущем цикле не удалось достичь цели по количеству стабильных пар (≥3), что указывает на недостаточную надежность сигналов, особенно по 30-дневным и 365-дневным периодам.
2. **Слабая корреляция между таймфреймами**: В большинстве случаев наблюдается несогласованность между таймфреймами (например, 0 из 4 таймфреймов за рост в AUDNZD и GBPNZD), что может указывать на отсутствие четкого тренда.
3. **Низкий винрейт**: Винрейты по многим парам находятся на грани приемлемости, что может привести к увеличению числа убыточных сделок.

# Рекомендации
1. **Увеличить STRONG_CONFIDENCE**: Установить значение на **92** (вместо 88) для повышения уверенности в сигналах.
2. **Увеличить STRONG_RATIO**: Установить значение на **0.65** (вместо 0.55) для повышения доли сильных сигналов.
3. **Увеличить STRONG_ADX_H1**: Установить значение на **30** (вместо 25) для фильтрации слабых трендов.
4. **Увеличить STRONG_ADX_H4**: Установить значение на **25** (вместо 20) для повышения надежности сигналов на более длительных таймфреймах.
5. **Увеличить STRONG_PERSISTENCE**: Установить значение на **85** (вместо 80) для повышения требований к последовательности тренда.

# Что НЕ менять
- **STRONG_ADX_H1 и H4**: Не менять на более низкие значения, так как это может привести к пропуску сильных трендов.
- **Формат анализа**: Сохранить текущую структуру анализа (M15/H1/H4/D1), так как она уже показала свою эффективность в определении трендов.

# Риски
1. **Снижение количества сигналов**: Ужесточение фильтров может привести к уменьшению общего количества доступных сигналов, что может затруднить торговлю.
2. **Пропуск потенциально прибыльных сделок**: Более строгие пороги могут привести к пропуску сделок, которые могли бы быть прибыльными, но не соответствуют новым критериям.
3. **Увеличение времени ожидания**: Из-за более строгих фильтров может потребоваться больше времени для нахождения подходящих торговых возможностей, что может негативно сказаться на общей активности торговли.

## 🧠 Эвристический AI-обзор (без LLM)

### Метрики
- **WR за последние 5 часов:** —% (0 решений)
- **WR на 28-парном бэктесте:** —%
- **Деградировавшие пары:** EURCHF

### Диагноз
- WR в норме — текущая жёсткость работает.
- Пары в деградации (1): `EURCHF` — рассмотреть исключение из топа на 24ч.

### Рекомендации к параметрам
- В `app/cycle.py._select_strict` добавить временный blacklist для деградировавших пар: ['EURCHF'].

### Точки правки
- `app/cycle.py`: `STRONG_CONFIDENCE`, `STRONG_RATIO`, `STRONG_ADX_H1`, `STRONG_ADX_H4`, `STRONG_PERSISTENCE`, `PREMIUM_ADX_H1`, `PREMIUM_PERSISTENCE`, `MIN_PICKS`, `MAX_PICKS`
- `scripts/cycle_5h.py`: `MIN_TRADES_PER_DAY`, `TOP_N`
